# Phase 2：从场景展示升级为可玩 Demo

## 核心方向

玩家身份改为 `ARES BASE ALPHA` 火星巡检员。第一段体验是“基地氧气产线报警，玩家去氧气站检查，再恢复太阳能阵列供电”。目标是让 Demo 从“看基地”变成“我在基地值班”。

## 必做改造

1. 修正操作
   - WASD 以相机朝向为准。
   - 修正左右方向反转。
   - 修正人物本地 `-Z` 朝向和移动方向不一致。
   - 窗口失焦时清空按键，避免松手后继续移动。

2. 重做玩家角色
   - 用程序化低模做火星工程宇航员。
   - 包含头盔、深色面罩、胸前控制板、背包、氧气管、手臂、腿、靴子、手套。
   - 加简单走路动画：身体轻微上下起伏，手臂和腿反向摆动。

3. 重做火星车和机器人
   - 巡检车：六轮、传感器桅杆、车灯、货仓。
   - 维修机器人：小体型、状态灯、机械臂。
   - 货运车：低速巡逻，作为基地生活感元素。

4. 加游戏流程
   - 首屏：火星基地远景 + `ENTER BASE`。
   - 进入后显示任务：`前往氧气生产站`。
   - 靠近氧气站显示 `E 检查`。
   - 检查后氧气站灯光从红变绿，并推进任务到太阳能阵列。

5. 拆分工程
   - `main.ts` 只负责启动。
   - 拆出 world、player、controls、missions、interactions、ui。
   - 先不引入复杂状态管理库。

## 验收标准

- 桌面端 WASD 方向直觉正确。
- 角色不再像胶囊人，一眼能看出宇航员。
- 火星车一眼能看出是工程车辆，不只是小盒子。
- 玩家 30 秒内能理解当前目标并完成一次检查交互。
- `pnpm build` 通过，浏览器控制台无错误。

## 参考来源

- `Caravan SandWitch`：低多边形探索、无战斗、车辆和角色生活感。
- `Messenger`：标题页、小星球、任务闭环、靠近交互和完成反馈。
- 当前生成的三张概念图：用于角色、车辆、基地布局方向。
