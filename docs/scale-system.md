# 尺度系统

## 参考尺度

- 宇航员：按 1.8m 级别理解。
- 火星巡检车：按小型载人车理解，约 4.5-5m 长、2.5-3m 宽，至少能坐下 1 名宇航员。
- 居住舱：按早期基地模块理解，直径约 4-5m，长度约 10-13m。
- Starship 参考：公开资料里 Starship 系统约 120m 级，直径约 9m；只看上级飞船也约 50m 级。

## 游戏压缩

当前 Demo 已从圆盘地形改成真实球体地形。玩家、建筑、石头和车辆都先记录在一组局部地表坐标里，再投影到星球球面法线方向上。

真实 Starship 如果按完整比例放到小星球上，会占掉大半个星球。当前 Demo 采用“科学参考 + 小星球压缩”的方式：

- 角色缩小到 0.76 倍，让玩家视角里建筑、穹顶和飞船显得更巨大。
- 火星车按 4.8-5m 级载人巡检车表现。
- 居住舱按 12m 级模块表现，温室穹顶按早期基地大体量空间表现。
- 火箭作为 `Starship-class lander`，表现为大型登陆器，高度接近 40m 级视觉体量。
- 火箭直径保留“明显比车和舱体大”的感觉。
- 小星球本身是玩具化场景，不按真实地理尺度。

## 验收标准

- 星球必须是完整球体，首屏能看到球形轮廓，而不是圆盘边缘。
- 玩家行走时应贴着球面法线，镜头跟随局部地表方向。
- 宇航员看起来能坐进火星车。
- 火星车不再像玩具盒子，而像可载人的巡检车。
- 火箭明显是大型飞行器，不是小塔。
- 居住舱、温室、工厂都比宇航员大一个合理数量级。
- 首屏仍保持小星球完整入画。
