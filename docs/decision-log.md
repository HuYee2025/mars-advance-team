# 决策记录

## 2026-07-08

### 激光剑未获得时给获取地点提示

原因：

此前为避免打扰，激光剑在未获得前按 `I/J` 是静默 no-op。用户现在明确要求：用户还没有获得激光剑但按下快捷键时，要给出提示，并包含获取地点。

决策：

- 玩家未获得激光剑时，按 `I` 或 `J` 弹出短提示：完成远古巨树拱门的时空之门穿越，坠落回火星后即可获得。
- 提示带约 `2.2s` 冷却，避免长按或连续误触刷屏。
- 已获得后的逻辑不变：`I` 展开/收回，按住 `J` 举起，松开恢复。

### 足球奖励改为古树拱门门洞，虫洞白场不再空镜

原因：

用户要求删除“足球网”物体，让足球奖励不再依赖独立球门/球网，而是把球踢进远古巨树拱门门洞；同时指出虫洞白色粒子散开后会短暂出现没有宇航员的虫洞空间，且白色粒子开场不够震撼。

决策：

- 删除火星足球独立足球网/球门实体，不再生成或挂载 `createFootballGoal()` 模型。
- 足球奖励判定改为使用远古巨树拱门本地坐标：足球进入拱门门洞体积后触发奖励、冷却和重生。
- 首次把足球踢进古树拱门门洞仍解锁相机；后续每次进门洞奖励 `+100` 积分并把足球重置回出生点。
- 虫洞白场取消中文“这是……”字幕，`index.html` 和 CSS 不再保留该文字节点。
- 白色粒子过渡开始时就初始化同一个 `wormholeFall` 镜头和宇航员飞行姿态，但白场期间不推进坠落进度；宇航员在白色粒子虚空中漂浮约 `3s`，随后粒子快速四散，直接露出有宇航员的黑色虫洞镜头，避免出现无人空镜。

### 精简远古巨树拱门地面碰撞体

原因：

用户在古树巨洞下方截图指出仍有透明碰撞体，坐标读数约为 `X +042 / Z +065`；火星蜘蛛经常在这里被卡住。复核旧 collider 后，最近的碰撞点是 `远古巨树拱门根部`，其局部位置约 `localX 33 / localZ -43`，已经位于可见门体 footprint 之外。

决策：

- 删除旧的拱门“拱形轮廓投影到地面”的 rows collider，不再用上方拱顶形状生成地面阻挡。
- 删除 `localZ -43` 一排根部 collider，避免门洞下方和可见实体外侧出现空气墙。
- 新 collider 只覆盖左右门柱和可见低矮根脚的地面 footprint，门洞中心和实体外侧保持可通行。
- 以截图点 `X +042 / Z +065` 复算，最近的古树拱门 collider 已离开约 `26` 游戏单位以上，不再贴住玩家或蜘蛛。

## 2026-07-07

### 移除宇航员假阴影并加入蜘蛛触碰伤害

原因：

用户截图指出宇航员脚下仍有旧的半透明假阴影，不符合当前改回真实 shadow map 的方向；同时希望火星蜘蛛从单纯环境生物升级为有明确风险的暗面异常生物。

决策：

- 删除宇航员本地半透明圆形接触阴影和对应每帧更新逻辑，宇航员脚下不再额外绘制假的黑色软阴影。
- 火星蜘蛛发现地面状态的宇航员后进入攻击状态，眼点由绿色变红，并主动向玩家所在球面方向爬行。
- 红眼蜘蛛触碰玩家一次扣 `20 * damageFactor` 体能，并设短冷却避免单帧连扣；体能归零时调用现有居住舱重生流程。
- 飞行状态不受蜘蛛触碰伤害；激光剑可用且展开时，蜘蛛继续回避玩家，不触碰也不造成伤害。
- 缩放枪只把红眼蜘蛛加入可锁定目标；命中后放大或缩小模型，同时把其触碰伤害倍率同步到当前缩放倍率，效果过期后恢复默认。

### 大型物体阴影改为跟随式真实 shadow map

原因：

用户指出太阳下高大的飞船和建筑缺少明显投影，空间体量感不足。曾尝试使用半透明椭圆假投影层，但画面里会出现像透明障碍物的深色长条，且不够真实。

决策：

- 移除 `projectedWorldShadows` 半透明假投影层。
- 保留 Three.js `DirectionalLight` 真实 shadow map，并让太阳光源和 shadow target 跟随玩家附近区域。
- 扩大 shadow camera 的 `near/far` 和正交覆盖范围，优先覆盖玩家周边的高大飞船、建筑、远古巨树拱门、黑色方碑、NASA 火星车和近火巨型流星。
- 近火巨型流星主体已 `castShadow = true`，纳入真实阴影尝试；远处两颗小流星距离过远，不纳入常驻阴影覆盖。
- 继续控制 shadow map 尺寸，先以当前 `2048` 为上限观察帧率。

### 修正暗面残骸碰撞与蜘蛛重叠

原因：

用户截图反馈 `X -007 / Z -145` 前方仍有透明障碍，同时实际坠毁飞船残骸可以任意穿过；另一处截图显示多只火星蜘蛛叠在一起不动，只有打开激光剑后才像一团对象一样集体挪动。

决策：

- 透明障碍的根因不是残骸 collider 半径，而是坠毁飞船位于暗面后，旧逻辑仍用正面 `planetNormal(x,z)` 生成 collider，导致碰撞体被投射到错误半球。
- 坠毁飞船残骸 collider 改为从 `CRASHED_SHIP_SITE_NORMAL` 出发，按残骸局部切线偏移后再转换为圆形碰撞系统的 `center`；这样实际残骸有碰撞，错误半球不再出现空气墙。
- 残骸 collider 仍只覆盖可见船体核心，不恢复过度外扩的整体圆形阻挡。
- 火星蜘蛛动态 collider 半径增大到覆盖身体轮廓，玩家不能直接穿过蜘蛛身体。
- 每只蜘蛛增加个人暗面活动中心、目标分离和位置分离修正；蜘蛛之间保持距离，避免重叠成一个模型，也避免完全同步集体移动。

### 修正太阳遮挡、投影可见性和古树脚部阻挡

原因：

用户反馈远古巨树拱门脚下仍有类似透明障碍物的区域；太阳可以透过高大飞船看到；飞船落在地面的投影几乎不可见。

决策：

- 太阳 sprite 保留高 `renderOrder`，但恢复 `depthTest`，让飞船、建筑和星球地平线可以正常遮挡太阳，不再让太阳始终画在模型前面。
- 已取消半透明椭圆假投影，改由跟随式真实 shadow map 尝试解决飞船落影问题。
- 远古巨树拱门脚部和根部 collider 收紧到更贴近可见黑色实体，减少门洞脚下通行区域被圆形 collider 外沿误挡。

## 2026-07-06

### 远处太阳从几何球壳改为曝光式发光体

原因：

用户指出远处太阳能看出明显几何体构造，后续又指出太阳远看应该像人眼看到的高亮发光体，大多数时候基本看不到细节；上一版火花粒子和整体视觉不像太阳。

决策：

- 可见太阳继续作为 `sunBody` group 存在，保留雷达标记和缩放枪可锁定对象身份。
- 取消实体球核心、半透明 shell、plasma 球壳和远景 `THREE.Points` 火花粒子。
- 远景视觉主体改为 canvas 生成的曝光式 sprite：接近纯白的高亮光芯、外发光和很淡的镜头光芒；不显示表面火纹或可数粒子。
- 增加单独的近景表现层。太阳被缩放枪放大到 `2x` 和 `3x` 时仍只放大默认发光太阳，不叠加近景贴图；到接近 `4x` 时才用默认太阳白色光盘作底，并在上方覆盖 `src/assets/sun-close-detail-texture.png` 透明近景太阳图。近距图显现时远景小太阳层同步退场，不再保留远景射线或额外大光环。
- 真实太阳方向、光照强度、太阳能阵列追日、暗面蜘蛛避光逻辑不变，只替换远处可见太阳的表现层。

### 缩放枪放大太阳改为清晰近距太阳

原因：

用户指出放大后的太阳画面仍然朦胧、不清晰；近距离观察时应直接使用参考太阳照片裁切出的主体，不能继续使用远景光晕。

决策：

- `src/assets/sun-close-detail-texture.png` 改为用户提供的圆形透明太阳主体图，只保留太阳圆盘和贴身亮边，不带周边日冕喷溅。
- 近距太阳直接使用该透明 PNG 的 `SpriteMaterial + AdditiveBlending` 呈现，模拟轻量 HDR 发光；贴图后方复用默认太阳白色光盘纹理作为底层背光，尺寸略大于贴图，用来填补边缘漏光和统一轮廓亮度。不再依赖黑底加法混合、自动抠图或额外圆形 alpha 蒙版。
- `4x` 近景太阳曾尝试 canvas 动态火纹层；因画面出现灰点且会增加纹理重绘开销，现已移除，保留默认白色光盘底层和近景太阳贴图。
- 近距太阳隐藏远景射线、大面积朦胧光晕、均匀圆环和额外 procedural 日冕层，避免再次出现“太阳不亮、只有外圈发光”的效果。
- 近距太阳贴图不做自转；太阳火焰可以轻微随整体 pulse 呼吸，但不能像风车一样围绕太阳旋转。
- 太阳缩放倍率同时影响场景太阳方向光和全屏过曝层；`2x` 开始明显增强，接近 `4x` 时模拟火星表面亮到难以直视。真实物理上辐照度随距离近似服从反平方律，本项目这里采用缩放枪玩法的线性近似，优先保证玩家能感知倍率。
- 当太阳放大到 `3x` 以上时，体能消耗按当前太阳倍率加速；对话、室内、乘车、电梯等原本暂停体能消耗的状态也生效，直到太阳恢复默认大小。

### 火星蜘蛛改为围绕古树动态避光

原因：

接入真实太阳时间后，原先固定的蜘蛛活动区可能在某些真实时间处于亮面，和“蜘蛛不喜欢太阳光”的设定冲突。用户还指出当前腿部动画像颤抖行进，不像爬行；新增的蜘蛛区熔岩/粒子云近看像平面贴图，没有真实裂缝体积。

决策：

- 蜘蛛活动中心不再固定在旧坐标，而是以远古巨树拱门为中心，根据当前真实太阳方向计算古树背光侧。
- 蜘蛛在当前位置被太阳照到时，会向古树暗面侧迁移；平时只在暗面侧小范围巡游。
- 腿部动画改为左右四组交替步态，移动时增加抬腿、前后伸展和身体轻微起伏，减少原来的原地颤抖感。
- 取消蜘蛛区贴地熔岩/粒子云 shader。后续若重做裂缝，必须用真实几何裂缝或局部地形下陷，不再用大面积发光平面贴图。

### Shader 世界文章只作为特效参考

原因：

用户提供技术文章 `Three 学习资料/一个 Shader 生成整个世界：Three.pdf`，文章解析了一个只用全屏 `ShaderMaterial`、fragment shader、`Ray Marching`、`SDF`、`fBm` 噪声、shader 内粒子、体积辉光、距离雾和 film grain 生成熔岩世界的 Three.js 案例。该方向和本项目的虫洞、传送门、沙尘、太阳和暗面异常特效有关，但本项目主世界还依赖实体模型、碰撞、雷达、任务、照片截图和多人同步。

- 后续可在开发模式加入 shader 参数调试面板，但上线前默认隐藏或移除。

### 移动端对话改为纯对话模态

原因：

手机和触摸屏空间有限，原方案让对话面板、角色头像、雷达、任务提示和移动按钮互相挤压，容易出现角色脸部被遮挡、互动按钮不清晰的问题。

决策：

- 移动端和触摸屏进入对话时，隐藏 HUD、任务条、雷达、标签、摇杆、移动按钮和缩放枪瞄准界面。
- 对话保留全屏模糊背景，只展示当前说话角色、对话文本、选项或“继续”按钮。
- 角色仍按原来的左右站位显示；谁说话就显示谁，不说话的一方隐藏，不再额外放大。
- 当前任务从雷达面板中移出，独立显示在游戏界面顶端居中。
- 操作指南暂时从界面中隐藏。

### 触摸端互动菜单改为底部抽屉

原因：

小屏幕上 2 个或以上互动按钮会和摇杆、跳跃、加速按钮重叠，直接展开所有互动项会让点击目标不清晰。

决策：

- 触摸端靠近互动对象时，先显示一个固定在操作区上方的小型“互动”入口。
- 点击“互动”后进入临时选择状态，底部抽屉纵向列出全部互动项。
- 抽屉打开时，摇杆和移动端动作按钮压暗并停止接收事件。
- 用户点互动项后执行操作；点“收起”或抽屉空白区域可回到游戏操作状态。

### 两辆车辆改为同速大环线巡航

原因：

原运输车在机器人车库附近做小半径巡逻，建筑碰撞推出逻辑会让它靠近建筑后侧身慢速漂移；两辆车速度也不一致。

决策：

- `01 车辆 电动巡检车` 和 `02 车辆 运输车` 使用同一个巡航速度。
- 取消运输车的车库小环线，两辆车都改为沿星球 great-circle 大环线巡航。
- 两辆车只通过路线 offset 拉开距离，保持同速同向。
- 大环线方位选择避开基地密集建筑，建筑碰撞保留为兜底保护，不再作为常态行驶方式。

### 桌面端对话框降低高度并抬高角色露出

原因：

桌面网页模式下原对话框高度过大，文本和按钮之外留下明显空白；同时角色头像被面板遮住，只露出头顶，削弱对话表现。

决策：

- 桌面端对话框高度降低到约 168-196px 区间，接近“文本加按钮三行”的紧凑高度。
- 桌面端角色头像向上抬高，保证说话角色露出上半身。
- 移动端全屏对话规则保持不变。

### 飞船维护机器人移出飞船底座投影区

原因：

02 号飞船维护机器人虽然已在碰撞体外，但仍位于货运飞船底座/船体投影下方，玩家只能看到腿部，且难以接近交互。

决策：

- 飞船维护机器人不再只按飞船中心径向外移。

- 所有维修机器人安全点外推余量从建筑碰撞半径外 `1.15` 提升到 `2.4`，运行时判定阈值同步提高，降低视觉上贴近建筑的风险。
- 已用几何审计检查 15 个维修机器人，未发现仍被困在固定碰撞体内的机器人。

### 桌面端双选项改为 Q/E 单手操作

原因：

大屏幕同时出现两个互动或对话选项时，依赖鼠标或 `Enter` 会打断左手移动区操作；玩家希望能连续按 `E` 推进对白，并在二选一时仍保持单手完成。

决策：

- 桌面端无选项对白使用 `E` 推进，保留 `Enter` 兼容。
- 桌面端两个并列选项时，`Q` 直接执行左侧，`E` 直接执行右侧。
- 多于两个选项时，保留方向键切换，并允许 `E` 或 `Enter` 确认当前选中项。
- 触摸端点击规则不变。

### 雷达未知目标默认显示为问号

原因：

项目早期雷达机制已经把未知生物或未知物体用 `?` 表示；后来加入的 `O` 键未知点开关让玩家需要额外记忆，而且 Elon 和黑色方碑这类未知对象不应该直接在雷达上暴露名称。

决策：

- 取消 `O` 键显示未知点的功能。
- 游戏开始后，雷达直接同时显示已知设施和未知对象。
- 未知生物、未知残骸、黑色方碑和 Elon 未被发现或对话前在雷达上统一显示为 `?`；首次发现或对话后移除 `?`，改用对应分类圆点。
- Elon 作为未知智能体加入雷达未知对象列表，不新增世界悬浮标签。

### 桌面端常用键位向左手集中

原因：

玩家希望减少右手离开鼠标或左手离开 WASD 区域的次数。原头盔灯 `I` 和地图 `M` 都偏右，不适合单手连续操作；第一人称此前只能靠滚轮逐步缩近。

决策：

- 头盔灯从 `I` 改为 `R`。
- 地图从 `M` 改为 `F`，长按 `F` 进入全屏地图，右上角地图按钮同步显示 `F`。
- 新增 `C` 一键进入第一人称，相当于把相机直接缩到宇航员头部位置；退出第一人称仍可通过鼠标滚轮拉远。

### 缩放枪键位改为 X 与 Q/E

原因：

缩放枪原来用 `U` 开关瞄准、`=` 和 `-` 调整大小，键位偏右，不符合当前“左手集中操作”的方向。玩家希望瞄准后继续使用 `Q/E` 完成缩小和放大。

决策：

- 缩放枪开关瞄准从 `U` 改为 `X`，`U` 不再绑定缩放枪功能。
- 缩放枪瞄准态下，`E` 放大，`Q` 缩小。
- `=` 和 `-` 的缩放设定保留，作为备用操作。
- 瞄准态下 `Q/E` 优先控制缩放枪，不触发普通互动或二选项操作。

### 低氧时雷达高亮补氧点

原因：

氧气压力是当前 1.0 的核心生存反馈之一，但玩家低氧时只看到数值和文字警告，不一定知道最近的补氧点在哪里。任务目标和补氧目标需要清晰分色，避免和金币混淆。

决策：

- 氧气背包低于或等于 35% 时，雷达开始高亮可补氧地点。
- 居住舱补给柜、氧气生产站和登陆飞船补给舱显示为绿色无外发光圆点。
- 当前任务目标显示为黄色圆点，并带专属放大渐隐黄色雷达光圈；金币显示为更小的 `$` 金币样式且不发光。
- 氧气高于 35% 时，补氧点不额外高亮，避免常态雷达过度拥挤。

### 体能改为居住舱恢复型生存指标

原因：

体能原来主要跟加速短时消耗和自动恢复绑定，玩家很难感知它是一个需要管理的生存指标。当前希望体能一般不容易耗完，加速会带来明显成本，跳跃则符合火星低重力探索直觉，并让居住舱承担“休整点”的作用。

决策：

- 地表普通移动会消耗体能，正常走路时基础规则约为氧气百分比下降速率的 0.5 倍，并乘以 `1.75` 手感系数，让体能压力比早期版本更明显。
- 持续加速时体能消耗为走路 2 倍，氧气消耗为走路 1.5 倍。
- 跳跃移动体能消耗按走路 0.5 倍处理，作为推荐探索方式。
- 原地站立会缓慢恢复体能，但氧气继续下降。
- 体能低于或等于 20% 时提示回居住舱休整。
- 氧气补给点只补氧，不补体能。
- 进入或离开居住舱会补满氧气和体能。
- 体能耗尽时，Alex 在居住舱内重新同步，氧气和体能恢复满值。

### 出室内后临时使用人物正面镜头

原因：

从居住舱或飞船内舱出来时，普通跟随镜头会落在建筑或飞船主体一侧，导致建筑遮挡宇航员身体，玩家短时间看不清角色位置和移动方向。

决策：

- 从居住舱、温室或飞船内舱出来时，启动临时正面镜头。
- 临时镜头站在人物正前方看向 Alex，让主体建筑位于角色背后，不遮挡角色身体。
- 玩家离出口达到安全距离后，自动恢复普通第三人称跟随视角。
- 进入室内、重生或回标题时清除临时镜头状态。

## 2026-06-25：多人 v1 采用免注册公共大厅

原因：

用户希望不同人进入游戏时自动生成新的宇航员，彼此可见，任务相同且不需要注册。当前 1.0 是纯前端静态网页游戏，因此多人能力需要新增实时同步层，但不应破坏现有单人可试玩闭环。

决策：

- 多人 v1 默认进入公共房间 `public-mars-alpha`。
- 浏览器首次进入生成匿名 `playerId`，保存在 `localStorage`；同一浏览器刷新后继续使用同一身份。
- 不使用 IP 作为玩家身份主键。IP 只可作为服务器日志、限流或防刷辅助，因为同一网络、VPN、校园网和公司网络都会造成误判。
- 任务内容相同，但进度各自独立；一个玩家完成任务不会推进其他玩家。
- 同步内容只包括远程宇航员的位置、朝向、移动速度、飞行状态和室内/室外状态。
- 远程玩家不可碰撞、不可互动，不参与任务触发。
- WebSocket 服务不可用时，前端安静降级为单人体验。

实现：

- 新增 `server/multiplayer.ts` 作为 Node WebSocket 房间转发服务。
- 新增 `src/multiplayer.ts` 作为前端多人客户端。
- 新增 `src/multiplayer-protocol.ts` 共享消息类型。
- `package.json` 新增 `dev:server` 和 `dev:multi`。

### 远处太阳改为实体球

原因：

原来的太阳使用 canvas 渐变 Sprite，远处看像一团朦胧光斑，不符合当前希望看到明确太阳球体的视觉要求。

决策：

- 太阳视觉从 `THREE.Sprite` 改为 `THREE.SphereGeometry` 实体球。
- 保留一层很薄的外发光壳用于亮度，但主体轮廓必须清楚。
- 风暴时只降低太阳实体和外发光透明度，不再回到大面积雾状光斑。

### HUD 增加在线人数和 Alex 职位

原因：

多人 v1 已经能在同一公共房间显示远程宇航员，左侧人员指标需要让玩家知道当前同时在玩的人数；Alex 下方原来的“在线”容易和在线人数重复。

决策：

- 人员 HUD 中 Alex 下方显示职位 `巡航员`。
- 人员 HUD 增加 `在线人数`，格式为 `1人`、`2人`。
- 在线人数口径为本地玩家 `1` 加当前房间远程玩家数量；WebSocket 不可用时仍显示 `1人`。

### 互动菜单最多显示两个选项

原因：

大屏幕底部同时出现三个互动按钮时，第三个选项会落到下一行，玩家无法用当前 `Q/E` 双键机制稳定选择，也容易遮挡画面。

决策：

- 同一位置的互动菜单最多显示两个选项。
- 显示前先按优先级排序：Mother 呼叫、进出建筑/升降梯、当前任务、机器人/福福、氧气补给。
- 超出两个的低优先级互动暂时隐藏；玩家离开重叠区域或条件变化后仍可重新出现。

### 黑色方碑埋入地面并允许缩放太阳

原因：

黑色方碑底部的圆形暗盘看起来像额外底座，不符合“方碑直接埋在火星土里、通过太阳投影产生压迫感”的方向；同时缩放枪既然是玩家获得的异常工具，远端实体太阳也应该能被瞄准并放大或缩小。

决策：

- 黑色方碑不再使用圆形接触盘或底座，保留直立黑色矩形主体和细边线，视觉上直接插入地面。
- 方碑与太阳之间的画面关系依赖场景光照、投影和背光构图，不再用假圆形阴影模拟压迫感。
- 太阳加入缩放枪可锁定目标，准星显示为 `太阳`。
- 缩放枪射程提高到能覆盖远端太阳实体；太阳缩放仍遵守当前 `0.25x` 到 `4x` 与 60 秒自动恢复规则。

### 缩放枪瞄准时锁定角色移动

原因：

缩放枪进入瞄准态后继续让 `WASD` 控制人物移动，会让玩家一边瞄准一边误走位，尤其是瞄远处太阳或黑色方碑时不稳定。当前键位方向已经强调单手操作，因此 `WASD` 在瞄准态应成为准星控制，而不是移动控制。

决策：

- 按 `X` 打开缩放枪后，Alex 停止移动；必须再次按 `X` 收起缩放枪后才能恢复走动。
- 缩放枪瞄准态下，`W/A/S/D` 改为控制准星方向：`A/D` 左右转向，`W/S` 上下调整。
- 鼠标锁定状态下仍可辅助瞄准，但键盘 `WASD` 是桌面端默认瞄准方式。
- 瞄准态下 `Q/E` 仍分别负责缩小/放大，不触发普通互动。

### 太阳视觉改为火球而非月亮

原因：

远处太阳改成实体球后，第一版颜色太淡、光晕太弱，看起来像一轮明月，不符合“太阳是大火球”的直觉。

决策：

- 历史方案：太阳保留实体球轮廓，表面改为黄白、橙红和深红混合的火纹贴图。
- 历史方案：外层增加橙红燃烧壳、日冕和更明显的外光晕。
- 历史方案：火球核心、外壳和日冕加入轻微旋转与闪烁，让太阳有燃烧感。
- 2026-07-06 后续更新：用户反馈几何体构造和火花粒子都不像太阳后，远处太阳改为曝光式发光 sprite，只保留柔和光晕和淡光芒；近距离或被缩放枪放大时才显示太阳表面细节。

### 奖励与随机金币系统 v1

原因：

当前任务完成、建筑探索和隐藏发现缺少即时正反馈。项目后续会继续增加任务，如果每个任务都单独写奖励逻辑，容易失控；同时用户希望先建立金币积累展示，为未来购物系统预留货币基础。

决策：

- 右上角 `F` 地图按钮左侧新增奖励 HUD，顺序为 `金币`、`积分`。
- 积分分为任务积分、探索积分和搞笑积分；主线默认 `+100`，支线默认 `+40`，建筑探索默认 `+10`，隐藏问号目标默认 `+30`，搞笑事件默认 `+10`，负面搞笑事件默认 `-10`。
- 奖励不写进对话或任务提示，只在玩家做到后即时发放，并配合数字放大、发光和音效。
- 用缩放枪放大或缩小太阳属于搞笑积分事件。
- 金币独立于积分，同一时间只存在一组随机金币，每组 10 枚，沿地表排成一列，按真实游玩时间 4 小时刷新或迁移。
- 金币视觉只保留金币本体，去掉每枚金币外面的圈。
- 金币生成必须避开 `world.colliders` 中的固体；雷达只显示一个金币组中心标记。
- 玩家穿过每枚金币时播放“叮”，金币从 3D 位置飞向右上角金币数值，到达后增加金币数。
- 氧气和体力不足导致的重生暂不扣积分、不扣金币。

### 第三人称防遮挡与升降台站立面修复

原因：

居住舱等大型建筑贴近玩家时，默认第三人称相机会落在建筑后方，画面可能被建筑模型挡满，玩家完全看不见 Alex。飞船升降台下落时，旧逻辑用手填 `surfaceY` 放置角色，没有读取平台地板的真实顶面，模型缩放和平台厚度会导致角色视觉上陷入平台。

决策：

- 室外第三人称相机增加基于 `world.colliders` 的视线遮挡检测；玩家到相机的连线穿过固体时，镜头沿地表法线平滑抬高，遮挡解除后自动回落。
- 第一人称、室内、飞船内舱和缩放枪瞄准态不启用该防遮挡抬镜头，避免干扰专用视角。
- 飞船升降台在 `ElevatorControl` 中记录真实地板对象 `surfaceObject`。
- Alex 站在升降台上时，以 `surfaceObject` 几何包围盒的顶面作为站立基准，并增加少量鞋底余量，避免上行或下行时陷入平台模型。

### 火星足球与轻量球面物理

原因：

需要在基地星球上放置一个一眼能看出是足球的可玩物体，并让 Alex、机器人、火星车和运输车碰到它时产生不同力度的滚动、飞行和反弹。项目当前没有完整刚体物理引擎，直接引入物理引擎会扩大复杂度和包体。

决策：

- 足球放在居住舱外的基地可见区域，作为可被玩家和活动物体推动的场景物体。
- 外观使用程序化 Canvas 贴图：白色球体、黑色足球拼块、缝线、明暗层次和 bump 细节，不新增外部贴图资产。
- 运动采用轻量球面物理：足球位置保存为星球法线、离地高度、地表切线速度和竖直速度。
- 竖直方向使用火星重力 `3.71 m/s²`；地表方向按 `PLANET_RADIUS` 转换为小星球表面角位移，并加入滚动摩擦和空气阻力。
- Alex 碰球时按 `playerVelocity` 的接触方向分量给冲量；加速跑会比普通行走踢得更远。
- 机器人、火星车和运输车碰球时用上一帧和当前帧的位置差估算实际速度；运输车/火星车的冲量高于维修机器人。
- 足球撞到建筑、飞船、岩石等静态 `world.colliders` 时按碰撞法线反弹并损失能量；撞到活动物体时按相对速度做动态反弹。
- 足球出生点穿过星球直径的另一面放置一个球门；优先使用严格对跖点，如果该点和已有碰撞体冲突，就在附近按切线圈搜索安全位置。
- 球门朝向足球出生点，进球判定使用球门本地宽度、高度和球网深度范围。
- 足球进入球网后每次奖励 `+100` 积分，随后立即重置到出生点，不保留在球网里；进球有短暂冷却，避免同一帧重复计分。

### 建筑探索自动计分与机器人外置巡逻

原因：

建筑探索属于“玩家已经到达”的事实，不需要再让玩家确认是否探索；额外按钮会干扰任务、进门和机器人通话。温室维修机器人仍然可能在视觉上陷入温室模型，说明机器人巡逻点和建筑碰撞体之间的安全余量不够。

决策：

- 建筑探索积分改为进入建筑互动范围后自动发放。
- 不再显示“探索 XX 建筑”互动按钮，也不再为建筑探索积分弹出浮动提示；右上角积分数值仍会更新。
- 当前任务互动仍可顺手触发建筑探索计分，但不要求玩家额外确认。
- `02 机器人 温室维修工` 的出生和巡逻点外移到温室正门外更远区域。
- 机器人巡逻点避障余量加大，避免数学碰撞体外但视觉上仍被建筑埋住或困住。

### 飞船出舱到升降梯落地前保持正面镜头

原因：

从飞船内舱出来后会启用人物正面镜头，但旧逻辑按玩家距离舱门原点释放镜头。升降梯下行时，玩家虽然还在平台上，和舱门原点的距离会快速变大，导致下行一半突然切回默认第三人称跟随视角，观感突兀。

决策：

- `exitFrontCamera` 仍可按距离释放，用于居住舱、温室和普通出舱后的移动恢复。
- 如果玩家正在乘飞船升降梯下行，正面镜头不按距离释放。
- 直到升降梯到达地面、`ridingElevator` 结束后，再恢复默认第三人称跟随镜头。

### 首页语言切换与游戏内锁定语言

原因：

游戏需要面向中文和英文玩家，语言入口应在首页第一屏可见。进入游戏后再切换语言会打断游戏状态和沉浸感，也可能让运行时按钮、任务文本与当前交互状态不同步，所以语言选择只在首页开放。当前项目文字散落在 HTML、HUD、任务提示、互动菜单、地图标签和对话数据中，需要先建立统一的轻量本地化管线。

决策：

- 右上角新增语言切换按钮，默认语言为 `简体中文`。
- 中文模式下按钮显示 `English`；英文模式下按钮显示 `简体中文`。
- 语言切换按钮只在首页显示；进入游戏后隐藏，且逻辑层会忽略新的切换请求。
- 语言选择保存到 `localStorage["mars.language"]`，刷新后保留上次选择；首次进入仍默认中文。
- 首页静态文案、HUD、任务条、奖励读数、地图按钮、互动菜单、退出确认、操作指南和缩放枪 UI 使用 i18n key 或静态文本扫描器刷新。
- 运行时任务、短提示、标签、地图显示和对话舞台通过 `localizeText()` 做英文显示；中文文案仍作为项目权威源文本保留。
- 长剧情对白后续可继续逐条补充更自然的英文精翻，不影响当前中文默认体验。
- 标签创建阶段只保存中文源文本，不调用语言系统；等 i18n 初始化完成后再统一刷新标签，避免页面加载阶段因初始化顺序导致脚本中断。
- 新增 `pnpm i18n:audit` 作为双语缺口检查。它会检查 HTML i18n key 的中英完整性，并把新增中文源文本和当前 baseline 对比，提示是否需要补英文词表或翻译表。
- 当前已有中文剧情和源文本写入 `scripts/i18n-audit-baseline.json`，避免历史存量噪音淹没真正新增的缺口；确认某批新增中文暂不翻译时，才运行 `pnpm i18n:audit:update` 更新基线。

### 碰撞触发探索积分

原因：

建筑探索原本只按交互点半径自动发放。飞船和大型建筑的可见模型、物理阻挡范围与交互点半径不完全一致，玩家可能已经撞到模型并被挡住，但还没有进入探索判定半径，导致积分不变。

决策：

- 保留进入建筑互动范围自动给分的逻辑。
- 在玩家与 `world.colliders` 中的建筑、飞船、太阳能阵列等实体发生首次碰撞时，也触发对应探索积分。
- 建筑碰撞复用原建筑探索记录，仍只奖励一次，并继续参与“全部建筑探索完成”统计。
- 飞船和太阳能阵列碰撞使用独立一次性探索事件，不影响建筑总完成数。

### 火星尺度扩大 50%

原因：

随着飞船、建筑、机器人、足球和球门等内容增加，原星球表面显得偏拥挤。如果只放大星球半径、不展开布局坐标，设施之间的实际地表距离几乎不会改变；如果把全部模型一起缩放或放大，又会破坏已有交互半径、碰撞和视觉比例。

决策：

- 新增 `WORLD_EXPANSION = 1.5`，`PLANET_RADIUS` 从 `88` 调整为 `88 * 1.5`。
- 基地布局展开系数从 `2` 调整为 `2 * 1.5`，保持设施相对方位关系，同时让设施间地表距离扩大 50%。
- 玩家出生点、足球出生点、补氧点和黑色方碑等非 `spread()` 坐标通过 `expandWorldCoordinate()` 同步展开。
- 建筑、飞船、机器人、车辆、足球、碰撞半径和交互半径保持原尺寸，不随星球半径放大。

### 建筑外壳碰撞补强与足球手感加速

原因：

居住舱是长条舱体，旧碰撞体只用一排圆形 collider 近似，正面和侧面存在视觉模型已挡住、物理 collider 没覆盖的漏区。Alex 可以从非门区穿入舱体，足球也会因为复用同一套 `world.colliders` 而滚进舱内消失。其他矩形建筑如果继续只用单圆，也会有类似风险。足球在火星扩大后显得滚动距离偏短，游戏感不足。

决策：

- 新增 `addFootprintColliders()`，用多排圆形 collider 覆盖长条舱体和矩形建筑外壳。
- 居住舱、温室、氧气生产站、甲烷燃料厂、机器人车库、科研舱、物资仓和医疗舱都改为更连续的足迹碰撞。
- 居住舱和温室在玩家进入室内状态时关闭外部外壳碰撞；室外进入仍通过互动按钮完成，不允许从模型表面直接走进去。
- 足球静态反弹继续使用 `world.colliders`，因此建筑外壳补强会同时阻止足球进入建筑模型。
- 足球玩家/车辆/机器人碰撞冲量乘以 `2.0`，最高速度从 `16.5` 提到 `33`，并降低滚动与空气阻力，让火星足球表现更夸张、滚得更远。

### 流星尾迹雾化与近火圆形轨道

原因：

旧流星尾迹使用 `THREE.Line` 三点线段，视觉上像一根长天线，而且对反向轨道的流星不一定跟随真实运动方向。后续雾团版本虽然去掉了线条，但主体过白、像自发光球，尾迹也像一个个光圈，不像火星车粉尘那种真实雾态。用户希望陨石本体是黑白灰石质，不自发光，只在受到光照时可见；尾迹更接近粉尘雾。

决策：

- `Meteor.trail` 从 `THREE.Line` 改为 `THREE.Group`，内部使用多个 `SpriteMaterial` 雾片作为尾迹，复用火星车粉尘的 `getDustFogTexture()`，不再使用发光球形光圈。
- 新增低精度陨石贴图 `src/assets/meteor-rock-generated.png`，由 Image 生成后压缩到 `128x128`，当前大小约 `26KB`。
- 陨石主体从 `MeshBasicMaterial` 改为 `MeshStandardMaterial`，使用灰黑贴图作为 `map` 和 `bumpMap`，不再自发光；远处只通过缩放和透明度弱化，近处靠场景光照显示坑洼石质。
- 每帧用当前轨道位置和前方采样点计算真实运动方向，尾雾沿运动反方向排列，避免尾迹方向错误。
- 三颗流星仍保留；两颗远处流星按太阳轨道上的远处天体处理，不参与缩放枪判断。
- 第 1 颗改为固定周期的绕火星圆形近火轨道，每现实约 `90` 秒绕行一圈；轨道半径恒定为 `PLANET_RADIUS + 150`，即高出火星地表约 `150` 游戏单位，约等于三艘飞船叠加高度；它从玩家初始基地上空轨道点开始，并每圈回到同一上空轨道点。该方案替代旧的“近火掠过高度曲线”，避免从玩家视角再次产生贴地或擦地平线飞过的错觉。
- 近火陨石本体使用不透明深灰石质材质，不再使用蓝色气体罩、蓝色气体尾巴或明显蓝色自发光；玩家必须直接看到石质陨石本体，不能再出现大蓝球观感。尾雾只保留给另外两颗远处流星。
- 近火流星加入雷达显示，使用蓝色流星点，并在普通雷达排序中优先保留；另外两颗远处天体不作为雷达重点目标。
- 远处两颗不掠过火星，始终保持在 `PLANET_RADIUS + 980` 以上的远距离天空区域，以 7 到 11 分钟左右的慢周期做随机椭圆漂移，像远处行星或小星体。

### 职位晋级系统绑定积分和正式任务

原因：

用户希望 Alex 从 `实习巡航员` 开始，并在完成任意任务后成为 `初级宇航员`；同时职位需要和积分系统建立关系，但金币不应干扰职位资历。

决策：

- 人员 HUD 中 Alex 下方不再显示固定 `巡航员`，改为动态职位。
- 初始职位为 `实习巡航员`。
- 完成任意正式任务，或积分达到 `100`，晋级为 `初级宇航员`。
- 后续职位按积分门槛晋级：`300` 正式宇航员，`700` 高级宇航员，`1500` 火星任务专家，`3000` 火星基地副指挥官。
- 最高称号 `第一位居民` 暂以主线完成且 Mother 信任、基地状态较高作为解锁条件，不靠刷分。
- 职位只升不降；负面搞笑扣分不会降级。
- 金币作为未来消费货币，不参与职位晋级。

### NASA Perseverance 火星车纪念点

原因：

用户希望火星表面出现现实中已经抵达火星的 NASA 火星车，增强真实感。NASA 官方资料显示 Perseverance 位于 Jezero Crater，任务与古河流三角洲/湖泊环境、取样和火星地质探索相关；外观上有六轮、桅杆相机、仪器箱和机械臂等高识别特征。

决策：

- 在游戏星球上新增静态地标 `NASA 火星车 Perseverance / Jezero Crater`。
- 模型使用 Three.js 几何体低多边形拼装，不引入外部 NASA 模型；重点保留六轮、摇臂支架、桅杆相机、车身仪器箱、后部 RTG 圆筒和机械臂/钻头。
- 它作为真实感纪念点和地图标签存在，不加入动态巡逻车辆系统，也不影响主线任务。
- 地点放在基地外侧空旷区域，作为 Jezero Crater 的游戏化代表位置，避免与现有建筑、飞船、足球玩法重叠。

### 神秘物体雷达与隐藏发现统一规则

原因：

雷达此前只有零散的 `unknown` 判断，黑色方碑等少数目标会显示问号，但足球、NASA Perseverance 火星车这类非基地常规设施没有统一归类。后续继续增加异常物、纪念物或外来物体时，如果每个目标单独写雷达和积分逻辑，容易出现显示不一致或遗漏奖励。

决策：

- 神秘物体统一包括足球、NASA Perseverance 火星车、黑色方碑、未知生命和坠毁/未编号异常物体。
- 基地建筑、飞船升降台、太阳能阵列、普通机器人、普通车辆、氧气补给点、金币和主线任务目标不归入神秘物体。
- 雷达上的神秘物体未发现前用 `?` 图标表示。
- 未发现前，展开雷达只显示“未知物体”或“未知生命迹象”，不泄露真实名称。
- 玩家首次接近神秘物体，或首次碰撞足球时，奖励隐藏发现积分；同一目标只奖励一次。
- 发现或对话后，雷达移除 `?` 图标，改用对应分类圆点；展开雷达显示真实名称，例如“火星足球”或 “NASA 火星车 Perseverance / Jezero Crater”。

### 基地亮面集中布局

原因：

火星放大后，主要建筑、飞船和太阳能阵列在首页视角里显得过于分散，不像一个工程基地。最初基地概念图的组织方式更接近“居住舱中心 + 温室/工业/车库/通信/飞船/太阳能阵列围绕”的紧凑组团，因此需要只移动大型基地设施，让首页亮面看起来像完整基地。

决策：

- 01-09 号主要建筑、01-03 号飞船和 A/B/C 太阳能阵列集中到首页亮面。
- 居住舱作为中心，温室和氧气站靠近生活核心，甲烷燃料厂、车库、科研舱、物资仓、医疗舱围绕外侧。
- 登陆飞船、货运飞船和返回飞船放在基地外围安全距离；01/02 飞船保持当前合理间距并作为一组远离太阳能阵列和医疗舱，03 返回飞船进一步远离基地和另外两艘飞船，保留升降梯和 Elon 支线空间。
- 太阳能阵列集中到亮面直射区域，三组平行排成一条赤道式电力场，间距一致，并远离货运飞船和医疗舱。
- 根据首页实际受光画面，太阳能阵列从背光侧旧位置 `(44,-70) / (64,-70) / (84,-70)` 先移到 `(48,54) / (66,54) / (84,54)`；后续根据截图修正为更靠近 03 返回飞船一侧的亮面区域 `(-22,52) / (-4,52) / (14,52)`，并同步更新连接管线到阵列 C 的端点。
- 同步更新基地管线和氧气补给点坐标，避免视觉连接与补给雷达指向旧位置。
- 坠毁飞船残骸和福福救援点不移动，继续留在暗面；黑色方碑、足球、球门、NASA Perseverance 火星车、岩石、金币刷新逻辑和彗星/陨石也不移动。
- 模型尺寸、碰撞半径和交互半径不变，只改变大型基地设施的位置。

### R 任务按钮与地图互斥规则

原因：

移动端屏幕空间有限，当前任务卡和互动面板、地图展开态容易互相遮挡。用户希望任务提示从固定卡片改成 `R` 按钮控制，并且在移动端 `R` 与 `F` 两个展开面板只能出现一个；桌面端空间充足，任务和地图可以同时打开。

决策：

- 游戏开始后，`V` 右侧显示 `R` 任务按钮，桌面端和移动端都通过它展开/关闭当前任务。
- 首次进入游戏时任务卡短暂展开后自动收起到 `R`，让玩家理解按钮用途。
- 有新任务提示时，`R` 外框发光；玩家打开任务卡后清除提醒。
- 移动端打开 `R` 时自动收起 `F` 地图，打开 `F` 时自动收起 `R` 任务卡。
- 桌面端允许 `R` 任务卡和 `F` 地图同时打开。
- 首页语言按钮右边缘与游戏内 `F` 按钮右边缘对齐，保持入口页和游戏 HUD 的右侧基准一致。

### 删除头盔灯/照明功能

原因：

`R` 已改为任务面板入口，移动端也需要减少右下角按钮负担。继续保留“照明/头灯”会增加按键冲突和 HUD 复杂度。

决策：

- 移除移动端“照明”按钮。
- 移除玩家头盔灯 SpotLight、地面光斑、开关状态和每帧更新逻辑。
- 移除多人同步协议里的 `lampOn` 字段和远程玩家头灯光源。
- 操作指南和技术方案不再声明任何头盔灯快捷键。

### 飞行模式降落与空中动画修正

原因：

作弊码飞行模式下，`Control/Ctrl` 下降被最低飞行高度卡住，无法真正落地；空中移动仍复用走路步态，看起来像在天上跑步；从高处降到飞船走廊或升降台区域时也缺少可行走落点判定。

决策：

- 飞行下降允许高度降到 `0`，到火星地表后自动关闭飞行模式、清除上升/下降键并隐藏飞行喷焰。
- 飞行中 Alex 不再播放走路摆腿，改为手臂和双腿轻微后摆，表现飞行背包推动时身体被气流拖住的姿态。
- 飞行下降时检测三艘飞船升降台顶部和高处廊道的 `walkBounds`；如果玩家落在可行走范围内，会吸附到该平台并切换为正常步行/升降台行走状态。
- 该落点吸附只在升降台到达顶部且未运动时生效，避免玩家在运动平台中途穿模。

### V 按钮开关态视觉统一

原因：

HUD 的 `V` 按钮原先在关闭信息时反而显示高亮，和 `F` 地图按钮“打开才有色”的视觉规则相反，容易让玩家误判当前状态。

决策：

- `V`、`R`、`F` 三个顶部圆形按钮关闭态统一使用半透明深色底。
- 对应信息打开时，按钮不填色、不加粗外圈，改用静态红色光晕表示打开状态。
- 只有 `R` 在有新任务信息时允许动态脉冲发光；打开任务面板后，动态脉冲停止，保留静态发光直到关闭。
- 顶部按钮禁用浏览器默认蓝色焦点框，焦点态也纳入红色光晕体系，确保只出现不发红光、静态红光和动态红光三种状态。

### 金币与积分 HUD 按钮化

原因：

右上角金币和积分原先是带中文标签的纯文本，和 `V`、`R`、`F` 三个顶部按钮的视觉体系不一致；移动端宽度也更紧张，需要减少文字长度。

决策：

- 金币和积分保留在 `F` 左侧，改为同 HUD 按钮体系的圆角矩形。
- 金币不显示“金币”文字，初始显示 `$`；获得金币后显示三位数字，例如 `001`。
- 积分不显示“积分”文字，初始显示 `0`；获得积分后显示三位数字，例如 `001`。
- 金币按钮颜色和左侧 HUD “基地”标题一致；积分按钮颜色使用顶部按钮红色光晕色系。
- 金币/积分按钮的边框透明度、深色底、胶囊圆角和字体厚度沿用首页语言切换按钮，但不改变金币黄和积分红。

### 剧情对话框响应宽度

原因：

大屏幕下剧情对话框接近横向拉满，阅读区域显得过宽，也压迫画面空间。用户希望宽屏约占屏幕宽度 70%，小屏幕可以更宽到 90%。

决策：

- 剧情对话舞台新增 `--dialogue-panel-width` 变量。
- 桌面和宽屏使用约 `70vw`，并通过 `clamp()` 保留最小可读宽度和左右安全边距。
- 小屏幕 `max-width: 700px` 时切换为约 `90vw`。
- 小屏幕对话框固定高度并向上抬，不再随内容收缩或撑开；正文区域最多三行，超过三行由运行时自动分页，选项/继续按钮固定跟在正文区域下方。
- `aria-pressed` 语义同步调整为表示“HUD 信息正在显示”，而不是表示“HUD 已折叠”。
- `V` 折叠态不再强制隐藏 `R` 任务面板，避免玩家点击发光的 `R` 后任务已打开但被 HUD 折叠规则遮住。

### 互动按钮窄宽度与圆角统一

原因：

移动端互动入口原先横向占位偏宽，在人物靠近左侧 HUD 区域时容易碰到或遮挡状态文字；桌面端互动选项也需要与当前 HUD 按钮体系更一致。

决策：

- 移动端互动入口改为居中窄宽度，按屏幕宽度响应式变化，最大不超过中等按钮宽度。
- 互动入口和桌面互动选项统一为轻微圆角矩形，保持蓝色描边和深色半透明底。
- 展开的移动端互动抽屉仍保留底部面板形态，但其中选项按钮沿用圆角矩形风格。

### 移动端双选项对话 A/B 模式

原因：

小屏对话框固定高度后，两个长选项纵向堆叠会挤占正文区域，第二个按钮容易掉出面板。用户希望按钮固定在一行，并列成两个短按钮。

决策：

- 移动端小屏对话只有两个选项时，正文自动追加 `A.` / `B.` 两行选项说明。
- 底部选择按钮固定一行两列，只显示 `A` 和 `B`，并通过 `aria-label` 保留完整选项含义。
- 桌面端和三个以上选项继续显示完整按钮文案，避免信息被过度压缩。

### 对话按钮音效反馈

原因：

剧情对话按钮原先只有视觉变化，点击后缺少和首页按钮一致的即时反馈。用户希望选择对白时发出与“进入基地”相同的短促滴声。

决策：

- 对话选项确认、继续翻页、推进对白和关闭对白统一调用 `playUiBeep()`。
- 音效放在对话动作函数内部，而不是只绑定鼠标点击，确保键盘、触摸和点击反馈一致。
- 无效确认不播放音效，避免玩家按错键时产生误导。

### ARES 火星历动态日期

原因：

网站首页已经上线，标题上的 “2050 年 / 第 30 火星日” 不应再作为静态装饰。用户希望它能随真实时间推进，同时保留硬科幻可信度。

决策：

- 采用 NASA Mars24 使用的平均火星太阳日长度 `88775.244` 秒作为一个 sol。
- 以 `2026-06-27 00:00 Asia/Shanghai` 为 ARES 基地民用历上线锚点，对应 `2050 年 / 第 30 火星日`。
- 之后每经过一个真实 sol，显示 sol 数 +1；火星日长于地球日，所以日期跳转点每天会比北京时间午夜晚约 `39分35秒`。
- 火星年显示采用硬科幻民用历规则：每 `669` 个 sol 进位一年。该值来自火星年约 `687` 个地球日，折算约 `668.6` 个 sol 后取整。
- 中英文首页日期共用同一套计算，中文显示 `{year} 年 / 第 {sol} 火星日`，英文显示 `Year {year} / Sol {sol}`。

### 首页语言切换改为国旗圆形按钮

原因：

首页右上角原本的文字语言按钮和游戏内 HUD 顶部圆形按钮体系不一致，视觉上显得像网页控件。

决策：

- 首页语言切换按钮改为与游戏内 `F` 按钮相同的位置、尺寸、圆形外框和 hover/focus 发光样式。
- 按钮不再显示 `English / 简体中文` 文字，改为圆形裁切的局部国旗图案。
- 中文界面显示美国国旗，表示点击切换到 English；英文界面显示中国国旗，表示点击切回简体中文。
- 国旗由 CSS 绘制，统一放大和裁切比例，优先展示左上角识别区域，避免新增图片资源。
- 后续视觉调整中，语言按钮进一步缩小为约 `24px` 的立体自转小球：保留点击逻辑，使用国旗 CSS 贴图、明暗高光和阴影模拟球体。

### 三次压缩后的新对话交接机制

原因：

长对话多次自动压缩后，容易出现上下文细节损耗、响应变慢和误读旧状态。项目已经采用“聊天只是过程，项目文件才是共同记忆”的协作原则，因此需要一个明确的触发点来检查记录完整性，并建议切换到新对话。

决策：

- 同一项目对话发生约 3 次自动压缩后，默认进入“交接检查”状态，不继续默认推进大型修改。
- 交接检查重点确认技术/UI 决策、重要取舍、剧情/设定/任务和当前实现状态是否已经写入项目文件。
- 新增 `docs/dev-log/current-handoff.md` 作为当前交接入口；若记录完整，建议开启新对话继续；若记录不完整，先补交接摘要或对应权威文档。
- 新对话恢复上下文时优先读取 `AGENTS.md`、`docs/project-management.md`、`docs/production-blueprint.md`、`docs/dev-log/current-handoff.md`、`docs/tech-plan.md` 和 `docs/decision-log.md`，不依赖旧聊天摘要。

### 体能与氧气探索节奏

原因：

用户希望火星探索更符合低重力直觉：加速更费体能和氧气，正常行走较稳，原地站立可以恢复体能，而跳跃应成为推荐探索方式。

决策：

- 正常地表行走时，体能基础下降速率约为氧气百分比下降速率的 0.5 倍；当前版本再乘以 `1.75` 手感系数，加快体能消耗。
- 加速移动时，体能消耗为走路 2 倍，氧气消耗为走路 1.5 倍。
- 跳跃移动体能消耗按走路 0.5 倍处理，让低重力跳跃成为推荐探索方式。
- 原地站立会缓慢恢复体能，但氧气仍按地表生命维持规则下降，避免等待恢复完全取消路线规划。
- 进入或离开居住舱时，体能和氧气都恢复到 100%；普通氧气补给点只更换氧气背包，不恢复体能。

### 英文版运行时文本精确翻译

原因：

英文模式原先依赖 `exactEnglishTexts` 和 `englishPhrasePairs` 混合处理。基础 HUD 可用，但剧情对白、机器人说明和部分运行时提示会被关键词替换成半中文半英文，影响英文版可读性。

决策：

- 新增 `runtimeEnglishTexts`，优先覆盖剧情对白、Elon 对话、机器人对白、缩放枪提示、建筑互动提示和常见 aria 默认文案。
- `localizeText()` 在英文模式下优先查 `runtimeEnglishTexts`，再查 `exactEnglishTexts` 和短语替换，避免完整句子被零散词片打碎。
- `scripts/i18n-audit.mjs` 纳入 `runtimeEnglishTexts` key，后续新增中文运行时文本若没有进入任一翻译表，仍会被审计提示。
- 本轮补全后，自定义本地化扫描确认英文模式下源码中文字符串转换后不再残留中文。
- 故事概要是独立静态页，不走游戏主入口 i18n。首页打开故事概要时改为携带当前语言 query；`public/story-overview.html` 读取 query 或 `localStorage["mars.language"]`，并用内置英文映射翻译正文、章节索引、alt 和 aria 文案。

### 相机奖励与照片墙下载

原因：

用户希望把“足球进球”扩展成一个更有生活感的奖励：首次进球获得相机，玩家可拍摄火星照片，并在居住舱照片墙查看。最初提出照片同时保存到居住舱墙上和本地桌面，但浏览器网页不能在无用户确认的情况下静默写入桌面。

决策：

- 首次把火星足球踢进球门后，一次性解锁相机；后续进球只发放常规进球积分。
- 相机不建模，按 `G` 打开/收起取景框；取景时人物不移动，`A/D` 转向，`W/S` 缩放，`E/Enter` 拍照。
- 拍照结果保存在运行时内存和居住舱内部照片墙，照片墙固定 `7` 个实体相框；超过 7 张时新照片替换最旧照片。
- 下载不自动写桌面，改为玩家在照片墙大图查看器中按 `E/Enter` 主动触发当前照片 PNG 下载，符合浏览器安全模型。
