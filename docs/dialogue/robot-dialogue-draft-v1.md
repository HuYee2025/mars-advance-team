# 机器人对白脚本草案 v1

更新时间：2026-06-24

用途：

这是机器人对白的写作草案，先不接入代码。目标是把当前“只说明设施用途”的单句对白扩展为 2-3 轮短对话，让机器人仍然保持执行单元气质，但信息更丰富，更有火星基地生活感。

写作原则：

- 机器人不是复杂人格角色，不发表哲学观点。
- 机器人可以有轻微风格差异：维修机器人像工单系统，货运机器人像物流系统，巡逻机器人像环境报告系统。
- 每段对白最好提供一个有用信息：设施功能、当前状态、风险、维护范围、火星环境逻辑。
- Alex 可以用简短问题引导，机器人回答保持机械、准确、略微笨拙。

格式：

- 触发：玩家靠近对应机器人并互动。
- 每个机器人 2-3 轮。
- `A` 代表 Alex，机器人使用实际标签。
