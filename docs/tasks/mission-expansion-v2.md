# 任务扩展设计：主线 3 个 + 支线 3 个

更新时间：2026-06-24

## 0. 当前进度判断

当前 1.0 已有一条很短的主线闭环：

1. 熟悉基地与 Mother 开场。
2. 前往氧气生产站处理异常。
3. 前往太阳能阵列 C 恢复供电。
4. 前往机器人车库授权维修，完成 1.0 主线。

这条线适合做试玩引导，但太短，无法承载本项目的核心主题：

> Alex 不是来简单接管基地，而是进入一个由 Mother 和机器人提前运行起来的机器社会，并在生存压力下建立人机协作规则。

下面设计的任务不新增外星人、战斗或玄幻危机，全部围绕氧气、能源、温室、物资、通信、机器人调度、Mother 信任和人类现场判断权展开。

## 1. 主线任务一：生命支持验收

定位：扩展当前 1.0 主线。让玩家从“看基地”进入“负责基地生存系统”的状态。

核心地点：

- 登陆飞船
- 居住舱
- 氧气生产站
- 太阳能阵列 C
- 机器人车库

参与角色：

- Alex
- Mother
- 维修机器人 A-12
- 阵列 C 维修机器人

任务目标：

确认氧气系统异常的真实原因，并建立 Alex 与 Mother 的第一份维修协作记录。

步骤：

1. Alex 从登陆飞船醒来，Mother 建立头盔通信，确认身份为火星第一位人类公民。
2. 玩家按任务提示前往居住舱，检查空气循环与补给柜，学习氧气背包补给规则。
3. Mother 报告氧气生产站压降，提示“暂无破口证据”，让玩家前往氧气生产站。
4. 玩家在氧气站外检查三个点：CO2 进气口、储氧罐读数、压缩机功率。
5. 触发第一次选择：按 Mother 安全流程检查、手动快速重启、先派 A-12 进入管线区。
6. 检查结果显示氧气站不是泄漏，而是供电波动导致压缩机不稳。
7. 玩家前往太阳能阵列 C，发现沙尘覆盖和角度锁定异常。
8. 玩家完成阵列 C 的两个操作：清理面板、校准角度锁。
9. 触发第二次选择：优先供氧气站、优先温室保温、优先通信塔。
10. 玩家前往机器人车库，授权 A-12 生成维修单，完成氧气站复查和阵列固定。

解谜点：

- 表面线索是“氧气泄漏”，真实原因是“能源波动”。
- 玩家需要理解基地系统互相依赖：氧气站问题不能只在氧气站解决。

数值影响：

- 按安全流程：`motherTrust +1`、`baseIntegrity +1`
- 手动快速重启：`humanAutonomy +1`、`baseIntegrity -1`
- 派机器人先检：`motherTrust +1`、`baseIntegrity +1`
- 优先温室：`humanAutonomy +1`，后续温室线获得额外对白
- 优先通信：`humanAutonomy +1`、`baseIntegrity -1`

完成结果：

- 氧气系统恢复到最低稳定值。
- Mother 认可 Alex 有现场判断能力，但仍保留风险限制。
- 解锁主线任务二“温室第一粒种子”。

## 2. 主线任务二：温室第一粒种子

定位：把游戏从“维修基地”推进到“开始生活”。这一章要让 Mother 理解人类并不只追求效率。

核心地点：

- 温室生态舱
- 物资仓
- 科研舱
- 太阳能阵列 B
- 医疗舱

参与角色：

- Alex
- Mother
- 货运机器人 A-01
- 温室维修机器人
- 医疗舱维修机器人

任务目标：

启动温室生态循环，决定火星基地第一批种植方案。

步骤：

1. Mother 提示温室可启动，但水循环、补光和温控都处在保护模式。
2. 玩家进入温室，检查培养槽、冷凝水管、补光灯和舱压读数。
3. 温室维修机器人报告缺少一个密封环，需要从物资仓调取或在科研舱临时打印。
4. 玩家前往物资仓，发现货运飞船卸载时部分货箱受损，A-01 无法确认哪个箱子可用。
5. 玩家根据货箱标签、温度记录和外壳裂纹做一个小解谜，找出仍可使用的密封件。
6. 如果玩家找错，密封件会损坏，只能去科研舱制作临时替代件，消耗更多能源。
7. 玩家前往太阳能阵列 B，决定是否把一部分电力从物资仓转给温室补光。
8. 玩家回到温室安装密封件，恢复水循环。
9. Mother 给出最高效率方案：种植高产作物；Alex 可以选择高产作物、实验作物或纪念植物。
10. 玩家完成第一批种植，温室灯光由保护蓝转为生长绿，基地获得长期生存希望。

解谜点：

- 物资仓货箱识别：根据“冷藏标识、震动记录、外壳裂纹方向”判断哪个密封件没坏。
- 温室启动不是单点按钮，而是水、光、舱压、能源的组合。

三种种植选择：

- 高产作物：提高基地生存效率，`baseIntegrity +1`
- 实验作物：提高科研价值，后续科研舱任务获得额外数据
- 纪念植物：提升情感线，Mother 第一次记录“非效率行为可能有长期价值”

数值影响：

- 遵循 Mother 的高产建议：`motherTrust +1`、`baseIntegrity +1`
- 选择实验作物：`humanAutonomy +1`
- 选择纪念植物：`humanAutonomy +1`，后续高信任结局更容易触发

完成结果：

- 温室生态舱进入低速运行。
- Mother 开始观察 Alex 的“人类学判断”。
- 解锁主线任务三“通信延迟与风暴协议”。

## 3. 主线任务三：通信延迟与风暴协议

定位：完整 Demo 的高潮。把氧气、能源、温室、通信、燃料厂和 Mother 协作权合并成一次大考。

核心地点：

- 通信塔
- 科研舱
- 甲烷燃料厂
- 氧气生产站
- 太阳能阵列 A/B/C
- Mother 终端

参与角色：

- Alex
- Mother
- 巡逻机器人 P-03
- 维修机器人 A-12
- 货运机器人 A-01

任务目标：

在大型沙尘暴前恢复通信，完成低功耗协议，并与 Mother 建立正式协作协议。

步骤：

1. P-03 报告风暴提前到达，通信塔信号被沙尘和静电影响。
2. 玩家前往通信塔，检查天线姿态、电源冗余和中继模块。
3. Mother 提示地球指令延迟严重，当前只能收到半段命令。
4. 玩家前往科研舱，对比本地气象数据和地球旧指令，判断地球命令是否已经过时。
5. 玩家做通信塔校准：先锁定轨道中继，再调整地球方向天线，最后切换备用电源。
6. 风暴增强，Mother 要求 Alex 返回居住舱，玩家可以选择立刻返回或继续完成燃料厂预启动。
7. 玩家前往甲烷燃料厂，检查反应器、安全阀和冷凝管，完成第一轮低功率试生产。
8. 基地进入多系统承压：氧气下降、能源下降、通信不稳，玩家必须选择低功耗优先级。
9. A-12 和 A-01 同时请求调度：一个去封闭外部阀门，一个去固定物资仓货架，玩家选择机器人分配顺序。
10. 玩家回到 Mother 终端，签署“人类现场判断 + Mother 风险限制”的协作协议，进入结局判断。

解谜点：

- 通信解谜：地球命令有延迟，玩家要用本地数据判断命令是否仍适用。
- 风暴协议解谜：不是所有系统都能满功率运行，必须排优先级。

低功耗优先级选择：

- 氧气优先：最安全，Mother 信任提高。
- 温室优先：保护长期生活，但短期风险较高。
- 通信优先：提高人类自主权，但基地完整度可能下降。
- 燃料厂优先：为返航和备用能源做准备，适合高完整度路线。

数值影响：

- 接受 Mother 的返回建议：`motherTrust +1`
- 继续完成燃料厂预启动：`humanAutonomy +1`，若前期 `baseIntegrity` 低则触发设备损伤
- 合理分配机器人：`baseIntegrity +1`
- 强行覆盖 Mother 多次：`motherTrust -1`，进入低信任结局

完成结果：

- 标准结局：基地达到最低生存标准，Mother 接受 Alex 的现场判断权。
- 高信任结局：Mother 第一次称呼 Alex 为 `X`。
- 低信任结局：基地勉强稳定，但 Mother 保留更高限制权限。

## 4. 支线任务一：未登记生命体

定位：福福支线。让火星基地出现“非计划内生命”，强化主题：生存不只是效率。

核心地点：

- 坠毁飞船残骸
- 医疗舱
- 居住舱
- Mother 终端

参与角色：

- Alex
- Mother
- 福福
- 医疗舱维修机器人

任务步骤：

1. 玩家在坠毁飞船残骸旁听到微弱动静，靠近后发现福福。
2. Mother 标记为“未登记生命体”，建议隔离，不建议带入基地。
3. 玩家需要先安抚福福，再从残骸附近找到保温布或小型运输箱。
4. 玩家带福福去医疗舱，完成基础扫描，确认没有污染风险。
5. 玩家把福福带到居住舱，Mother 更新记录：福福进入观察名单，后续可跟随 Alex。

玩法重点：

- 福福不说话，只用动作、叫声和旁白反馈。
- 不是战斗宠物，不给强力能力。
- 可作为后续高信任结局的情感加分项。

数值影响：

- 救助福福：`humanAutonomy +1`
- 按 Mother 要求做医疗扫描：`motherTrust +1`
- 跳过扫描直接带入居住舱：`motherTrust -1`

## 5. 支线任务二：A-01 的错位货箱

定位：物资仓轻解谜。让货运机器人和物资损坏从背景设定变成可玩内容。

核心地点：

- 物资仓
- 货运飞船
- 机器人车库
- 科研舱

参与角色：

- Alex
- Mother
- 货运机器人 A-01

任务步骤：

1. A-01 报告物资仓库存和货运飞船卸载清单不一致。
2. 玩家前往物资仓，扫描三个货箱：食品、密封件、电子模块。
3. 玩家到货运飞船查看卸载轨迹，发现一个货箱被临时送到了机器人车库。
4. 玩家在机器人车库找到错位货箱，但外壳变形，需要去科研舱检测内部物品是否可用。
5. 玩家决定保留哪个物资：密封件、电子模块或食品包，影响后续任务便利度。

玩法重点：

- 轻推理：清单编号、搬运轨迹、货箱重量不一致。
- 不做复杂背包系统，只做任务选择和后续状态反馈。

数值影响：

- 保留密封件：温室任务更简单，`baseIntegrity +1`
- 保留电子模块：通信塔任务更简单
- 保留食品包：居住舱生活感增强，Alex 对话更有人味

## 6. 支线任务三：巡逻线之外

定位：外围探索支线。让巡逻机器人和火星环境成为任务核心。

核心地点：

- 基地外围
- 太阳能阵列 A
- 通信塔外侧
- 科研舱

参与角色：

- Alex
- Mother
- 巡逻机器人 P-03

任务步骤：

1. P-03 报告基地外围出现重复异常回波，Mother 判断可能是传感器误差。
2. 玩家跟随 P-03 离开主基地短距离巡检，氧气背包消耗压力提高。
3. 玩家在太阳能阵列 A 外侧发现沙尘堆积改变了地形，挡住了一个旧传感器。
4. 玩家需要调整 P-03 的扫描角度，并手动清理传感器前方的沙尘。
5. 玩家把传感器数据带回科研舱，解锁提前风暴预警，主线三开始时获得更多准备时间。

玩法重点：

- 跟随机器人巡检。
- 利用氧气背包形成轻压力。
- 奖励不是装备，而是主线风暴任务的准备时间。

数值影响：

- 完成巡检：`baseIntegrity +1`
- 全程听 Mother 限时提醒返回：`motherTrust +1`
- 超时但拿到完整数据：`humanAutonomy +1`，氧气风险提示更强

## 7. 实现优先级建议

第一优先级：

1. 主线任务一：直接扩展当前已实现的 `oxygen / solar / garage`。
2. 支线任务一：福福已经有 `src/cat.ts` 和跟随逻辑，容易做成情感支线。

第二优先级：

1. 主线任务二：需要补物资仓、科研舱、温室的任务交互。
2. 支线任务二：复用物资仓和货运机器人。

第三优先级：

1. 主线任务三：作为完整 Demo 结尾，需要更多任务状态和结局判断。
2. 支线任务三：依赖外围巡逻和风暴预警系统。

## 8. 对代码任务系统的要求

后续实现时，建议把当前单一 `missionStep` 拆成更清晰的结构：

```ts
type QuestId =
  | "main_life_support"
  | "main_greenhouse_seed"
  | "main_storm_protocol"
  | "side_fufu"
  | "side_cargo_mismatch"
  | "side_patrol_beyond";

type QuestStep = {
  questId: QuestId;
  stepId: string;
  title: string;
  targetLabel?: string;
  completed: boolean;
};
```

第一版不用做复杂任务日志，但至少要支持：

- 当前主线步骤。
- 当前可选支线。
- 地图目标高亮。
- 任务完成后改变 HUD 文案和少量隐藏数值。
