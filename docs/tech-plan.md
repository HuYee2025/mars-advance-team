# 技术方案

## 技术栈

- Vite：轻量开发服务器和构建。
- TypeScript：保证场景代码可维护。
- Three.js：WebGL 渲染、程序化模型、灯光、相机和动画。
- ws：多人 v1 的轻量 WebSocket 同步服务。

## 渲染方案

- 程序化地形：径向网格生成弧面火星地表，顶点色制造沙丘层次。
- 运行时图片资源：火星地表贴图和对话立绘优先使用 WebP，控制首屏和对话加载体积。
- 程序化建筑：Cylinder、Box、Sphere、Cone 等基础几何组合出基地。
- 动态元素：机器人沿路径巡逻，基地灯光轻微闪烁，粒子表现沙尘。
- 车辆地表特效：火星车使用贴地半透明轮迹和低透明度细雾粒子表现尘土，不做实时地形变形。
- 标签系统：建筑对象投影到屏幕坐标，玩家靠近时显示中文标签。

## 性能目标

- 桌面端接近 60fps。
- 移动端至少 30fps。
- 第一版避免重型后处理、外部高模、复杂物理和实时阴影滥用。
- 后续如果增加模型，应优先使用 glTF/GLB，并控制贴图尺寸。

## 移动端限制

- 小屏幕只保留必要 HUD。
- 虚拟摇杆用于移动，右下角保留大号“跳”触摸按钮，大按钮左侧保留小号“加速”触摸按钮，并保留小号“照明”按钮；鼠标锁定只在桌面启用。
- 触摸端靠近可交互对象时，直接弹出可点击互动菜单；不要求玩家寻找或模拟 `E` 键。
- 触摸端对话按钮和“继续”按钮必须可直接点击，不能只依赖键盘确认。
- 粒子数量、阴影分辨率和像素比已经做了保守限制。

## 当前 1.0 操作规则

- 移动：`WASD` 或移动端摇杆。
- 进入互动：桌面端靠近对象后按 `E`；触摸端靠近后点击弹出的互动菜单。
- 互动选择：同一位置最多显示两个最高优先级互动；桌面端用 `Q` 执行左侧、`E` 执行右侧；触摸端直接点击菜单按钮。
- 对话选择：桌面端无选项时用 `E` 推进；两个并列选项时用 `Q` 选择左侧、`E` 选择右侧；多选项仍可用左右键切换、`E` 或 `Enter` 确认；触摸端直接点击对话按钮或“继续”。
- 跳跃：桌面端没有互动选择时按空格；触摸端点击右下角大号“跳”按钮。
- 地图：`F` 或右上角圆形 `F` 按钮；长按 `F` 进入全屏地图。
- 小屏地图：点击雷达进入全屏地图。
- 雷达：游戏开始后直接显示已知设施和未知对象；未知生物、未知残骸、黑色方碑和 Elon 统一显示为 `?`，不再需要 `O` 键开启。
- 雷达补氧提示：氧气背包低于 35% 时，居住舱补给柜、氧气生产站和登陆飞船补给舱显示为放大的橙色发光点，形式与绿色任务目标一致。
- 体能：地表移动会缓慢消耗体能，持续加速或连续跳跃会增加消耗；加速跑每秒消耗等于一次跳跃消耗；体能低于 20% 时提示回居住舱休整；体能耗尽时在居住舱内重新同步。
- 头盔灯：按 `R` 开关，替代旧的 `I` 键。
- 视角：按 `C` 在第一人称和默认第三人称之间切换；使用 `=` 和 `-` 调整镜头远近后，按 `0` 一键恢复默认第三人称视角。
- 星球尺度：火星视觉半径使用 `WORLD_EXPANSION = 1.5` 放大 50%，布局坐标同步展开 50%；建筑、飞船、机器人、车辆、足球等模型尺寸和交互/碰撞半径不随星球放大，避免基地显得过度拥挤。
- 语言：首页右上角提供语言切换按钮，默认 `简体中文`。中文模式按钮显示 `English`，点击切换为美式英文界面；英文模式按钮显示 `简体中文`，可点击切回。进入游戏后隐藏并禁用语言切换，当前局语言沿用首页选择。首页静态文案、HUD、任务、奖励、互动、地图、系统提示和对话舞台通过轻量 i18n 管线刷新。
- 双语检查：新增 UI、任务、对白或标签后运行 `pnpm i18n:audit`。脚本会强检查 `index.html` 的 i18n key 是否中英齐全，并基于 `scripts/i18n-audit-baseline.json` 提醒新增的中文源文本是否还未进入 `i18n`、`exactEnglishTexts` 或 `englishPhrasePairs`。当前存量中文已写入 baseline；确认新增中文暂时允许保留为中文权威源时，运行 `pnpm i18n:audit:update` 更新基线。需要让警告阻断时使用 `pnpm i18n:audit:strict`。
- 出室内镜头：从居住舱、温室或飞船内舱出来时，镜头临时切到人物正面，等玩家离开主体建筑遮挡范围后自动恢复跟随视角。
- 飞船升降梯镜头：从飞船内舱出来后，如果玩家启动升降梯下行，出舱正面镜头保持到升降梯到达地面停止，再恢复默认第三人称跟随，避免下行中途突然切镜头。
- 第三人称防遮挡：室外第三人称下，如果玩家到相机的视线穿过建筑、飞船或其他 `world.colliders` 固体，镜头会沿地表法线平滑抬高，离开遮挡后自动回到原高度，避免角色被居住舱等大型模型完全挡住。
- 飞船升降台：角色站立高度以升降台地板模型的实际顶面为准，并保留少量鞋底余量；不要再只依赖手填 `surfaceY`，否则平台运动和模型缩放会让角色陷入平台。
- HUD 显示：左上角圆形 `V` 按钮。
- 退出确认：游戏中按 `Esc` 打开确认窗口，默认选中“返回游戏”；切换到“退出游戏”并确认后才返回主页。
- 操作指南：按住 ESC 下方的 `· / ~` 键显示，松开隐藏；玩家使用过一次后，雷达下方的操作指南提示隐藏。
- 神秘代码：输入 `HUYEE` 播放“叮咚”音并切换飞行模式；飞行模式下按住空格上升，按住 `Control/Ctrl` 下降，WASD/方向键继续控制移动，再次输入 `HUYEE` 关闭。
- 缩放枪：程序化手持模型默认隐藏；玩家靠近黑色方碑按 `E` 触发对话并点击“获取”后装备到 Alex 右手。拥有后按 `X` 进入/退出瞄准态，中央显示缩放枪准星和锁定目标；瞄准态下 Alex 不能走动，`WASD` 改为全方位控制准星，按 `E` 放大、按 `Q` 缩小，`=` 和 `-` 保留为备用缩放键；效果按原始尺寸限制在约 `0.25x` 到 `4x`，60 秒后自动恢复。太阳也属于可锁定缩放目标，因此缩放枪射程需要覆盖远端太阳实体。
- 奖励 HUD：右上角 `F` 地图按钮左侧显示 `金币` 和 `积分`，金币在前、积分在后。积分和金币变化时数字弹跳发光，任务积分播放上升调奖励音，金币逐枚拾取时播放“叮”。
- 积分系统：前端运行时维护 `scorePoints`、`awardedEvents` 和建筑/隐藏发现记录。主线任务默认 `+100`，支线任务默认 `+40`，建筑探索默认 `+10`，隐藏问号目标默认 `+30`，搞笑事件默认 `+10`，负面搞笑事件默认 `-10`。建筑探索积分到达范围后自动发放，不显示确认按钮或浮动提示；如果玩家首次撞到建筑、飞船或太阳能阵列等实体碰撞体，也会立即触发对应的一次性探索积分，避免交互点半径和模型阻挡范围不一致导致漏判。后续新增任务应优先走统一奖励 API，只有特殊事件才单独覆盖分值。
- 职位系统：人员 HUD 中 Alex 下方显示动态职位，初始为 `实习巡航员`。前端运行时维护 `currentRank`、`completedAnyTask` 和 `rankRules`；完成任意正式任务或积分达到 `100` 后晋级 `初级宇航员`，后续按 `300 / 700 / 1500 / 3000` 积分晋级。职位只升不降，金币不参与晋级；最高称号 `第一位居民` 暂由主线最佳结局条件解锁。
- 随机金币：前端运行时维护单个金币组，每组 10 枚并沿地表切线排成一列，刷新周期为真实游玩时间 4 小时。生成时在星球表面随机抽样，并根据 `world.colliders` 避开建筑、飞船、太阳能阵列、石碑、岩石等固体；长时间无人收取时迁移到新位置，不叠加多组。雷达只显示金币组中心的一个金币标记。
- 火星足球：居住舱外放置一个可碰撞足球。足球使用程序化白色足球贴图、黑色拼块、缝线和 bump 细节；运动采用轻量球面物理，竖直方向使用火星重力 `3.71 m/s²`，地表方向按小星球切线滚动。Alex、机器人、火星车和运输车会按接触速度给足球冲量；建筑、飞船、岩石等 `world.colliders` 固体会让足球反弹并损失部分能量。足球出生点穿过星球直径的另一面放置球门，落点会避开障碍物；足球进入球网后每次奖励 `+100` 积分，并立即回到出生点。

## 部署方案

当前 1.0 的单人体验仍是纯前端静态网页游戏，构建后产物在 `dist/`。多人 v1 额外需要一个常驻 Node WebSocket 服务，用于同步公共大厅中其他玩家的位置和外观状态；该服务不保存任务进度，不需要数据库。
公开故事页图片位于 `public/story-assets/`，线上部署时使用 WebP 版本，避免复制未引用的大 PNG 副本。

推荐部署方式：

- 本地执行 `pnpm build` 或 Codex 内置 pnpm 构建。
- 上传 `dist/` 内容到服务器静态站点目录。
- 使用 Nginx 托管根路径。
- 如启用多人模式，运行 `pnpm dev:server` 或线上 Node 进程启动 `server/multiplayer.ts`，并让 Nginx 将 `/ws` 反向代理到该进程。

详细步骤见：`docs/deployment.md`

## 多人同步方案

多人 v1 采用“客户端权威位置广播 + 服务端房间转发”：

- 公共房间固定为 `public-mars-alpha`。
- 前端身份存储：
  - `localStorage["mars.playerId"]`
  - `localStorage["mars.displayName"]`
- 前端通过 `src/multiplayer.ts` 连接 WebSocket：
  - 开发环境默认：`ws://127.0.0.1:8787/ws`
  - 线上默认：同域名 `/ws`
  - 可用 `VITE_MULTIPLAYER_WS_URL` 覆盖。
- 客户端约每秒 12 次发送本地玩家快照：位置、四元数、速度、飞行状态、头盔灯状态和室内/室外状态。
- 服务端 `server/multiplayer.ts` 只维护在线玩家列表、心跳超时和状态广播；不判定任务、不写数据库、不保存长期进度。
- 左侧人员 HUD 通过多人客户端的远程玩家列表显示在线人数，最低显示 `1人`。
- 远程玩家复用 `createMarsEngineer()` 生成宇航员模型，并在本地做插值平滑。
- 远程玩家不可碰撞、不可互动，不进入任务系统。
- 当 WebSocket 服务不可用或断开时，前端安静重试，单人游戏继续运行。

## 音频方案

背景音乐使用原 MP3 的压缩版：

- 源文件：`星际低语_no-watermark (1).mp3`，约 10MB，只保留在本地，不提交 GitHub。
- 项目内文件：`src/assets/audio/mars-background-light.mp3`，64kbps，约 2.5MB。

原因：

- 保留原背景音乐的听感。
- 减小部署包体积。
- 保持纯前端静态部署，不增加后端依赖。
