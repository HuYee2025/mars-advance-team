# 技术方案

## 技术栈

- Vite：轻量开发服务器和构建。
- TypeScript：保证场景代码可维护。
- Three.js：WebGL 渲染、程序化模型、灯光、相机和动画。
- ws：多人 v1 的轻量 WebSocket 同步服务。

## 渲染方案

- 程序化地形：径向网格生成弧面火星地表，顶点色制造沙丘层次。
- 运行时图片资源：火星地表贴图和对话立绘优先使用 WebP，控制首屏和对话加载体积。
- 程序化建筑：Cylinder、Box、Sphere、Cone 等基础几何组合出基地。
- 动态元素：机器人沿路径巡逻，基地灯光轻微闪烁，粒子表现沙尘。
- 车辆地表特效：火星车使用贴地半透明轮迹和低透明度细雾粒子表现尘土，不做实时地形变形。
- 轨道星座：`ARES Starlink-M` 使用程序化低多边形卫星和微光点表现火星低轨通信星座，不引入外部模型或真实网络仿真。
- 标签系统：建筑对象投影到屏幕坐标，玩家靠近时显示中文标签。

## 性能目标

- 桌面端接近 60fps。
- 移动端至少 30fps。
- 第一版避免重型后处理、外部高模、复杂物理和实时阴影滥用。
- 后续如果增加模型，应优先使用 glTF/GLB，并控制贴图尺寸。

## 移动端限制

- 小屏幕只保留必要 HUD。
- 虚拟摇杆用于移动，右下角保留大号“跳”触摸按钮，大按钮左侧保留小号“加速”触摸按钮；不再提供“照明/头灯”按钮或快捷键；鼠标锁定只在桌面启用。
- 触摸端靠近可交互对象时，直接弹出可点击互动菜单；不要求玩家寻找或模拟 `E` 键。
- 触摸端对话按钮和“继续”按钮必须可直接点击，不能只依赖键盘确认。
- 粒子数量、阴影分辨率和像素比已经做了保守限制。

## 当前 1.0 操作规则

- 移动：`WASD` 或移动端摇杆。
- 首页：默认选中“进入基地”。`W/S` 或方向键上下切换按钮；`E`、`Enter` 或 `Space` 确认当前选中项；`Q` 直接进入“故事概要”；鼠标悬停可切换当前选中项，再用 `E` 或 `Enter` 确认。
- 进入互动：桌面端靠近对象后按 `E`；触摸端靠近后点击弹出的互动菜单。
- 互动选择：同一位置最多显示两个最高优先级互动；桌面端用 `Q` 执行左侧、`E` 执行右侧；触摸端直接点击菜单按钮。
- 对话选择：桌面端无选项时用 `E` 推进；两个并列选项时用 `Q` 选择左侧、`E` 选择右侧；触摸端直接点击对话按钮或“继续”。正式游戏对话最多只允许两个选项，匹配 `Q/E` 二选一操作；需要更多内容时拆成后续节点或改为单个继续按钮，不允许出现第三选项。
- 跳跃：桌面端没有互动选择时按空格；触摸端点击右下角大号“跳”按钮。
- 地图：`F` 或右上角圆形 `F` 按钮短按切换普通雷达；长按 `F` 进入临时全屏雷达，背景虚化、游戏世界继续运行，但键盘/人物操作焦点全部交给雷达，人物不可移动也不可触发普通互动；松开 `F` 后恢复到长按前状态。
- 全屏雷达缩放：按住 `F` 的全屏雷达状态下，`Q` 缩小雷达比例以显示更大区域，`E` 放大雷达比例以查看更小区域；放大时圆点和文字轻微变大。全屏雷达状态下 `Q/E` 优先用于雷达缩放，不触发互动选择。
- 小屏地图：点击雷达进入全屏地图。
- 雷达：游戏开始后直接显示已知设施和未知对象；未知生物、未知残骸、黑色方碑和 Elon 未被发现或对话前统一显示为 `?`，不再需要 `O` 键开启；首次发现或对话后移除 `?`，改用对应分类圆点并显示真实名称。常规分类用纯色小圆点区分，任务目标为黄色圆点并带一圈圈放大渐隐的黄色雷达光圈，氧气补给为绿色圆点，金币为更小的 `$` 金币样式且不发光，避免和任务引导混淆。雷达外圈内侧显示 `N/E/S/W` 方位环，随玩家视线左右旋转实时更新；坐标下方显示 `HDG 000°` 格式的朝向读数。
- 雷达补氧提示：氧气背包低于 35% 时，居住舱补给柜、氧气生产站和登陆飞船补给舱显示为绿色圆点；任务目标使用黄色脉冲圆点。
- 体能/氧气：地表行走时氧气按基础速率下降，体能基础消耗在“氧气数值下降速率 0.5 倍”的探索规则上再乘以 `1.75` 手感系数；按住加速移动时，体能消耗为走路 2 倍，氧气消耗为走路 1.5 倍；跳跃探索按走路 0.5 倍消耗体能，作为火星低重力下的推荐探索方式；原地站立会缓慢恢复体能但仍消耗氧气；进入或离开居住舱时氧气和体能都恢复到 100%。
- 头盔灯：已从当前版本移除，不再提供“照明”按钮、头灯快捷键或多人同步状态。
- 视角：按 `C` 在第一人称和默认第三人称之间切换；使用 `=` 和 `-` 调整镜头远近后，按 `0` 一键恢复默认第三人称视角。
- 首页镜头：首页展示火星时使用 `TITLE_PLANET_VIEW_SCALE = 0.7`，让标题页背景星球比游戏内实际星球视觉尺寸小 30%，减少首屏压迫感；游戏内星球半径和布局不受影响。
- ARES 火星历：首页火星日期不再是静态文本。采用 NASA Mars24 的平均火星太阳日 `88775.244` 秒作为一个 sol；以网站上线显示日 `2026-06-27 00:00 Asia/Shanghai` 对应 `2050 年 / 第 30 火星日` 为 ARES 基地民用历锚点。之后每经过一个真实 sol，火星日 +1；每 `669` 个 sol 进位一个 ARES 火星年。该规则参考 NASA GISS Mars24 算法说明和 NASA/JPL 火星任务资料中 “sol = 24h 39m 35.244s / 火星年约 687 地球日、约 668.6 sols” 的基准；游戏内显示采用整数 sol，故民用年按 669 sols 取整。
- 星球尺度：火星视觉半径使用 `WORLD_EXPANSION = 1.5` 放大 50%，布局坐标同步展开 50%；建筑、飞船、机器人、车辆、足球等模型尺寸和交互/碰撞半径不随星球放大，避免基地显得过度拥挤。
- ARES Starlink-M：火星半径按 `3390 km` 映射到当前 `PLANET_RADIUS = 132`，低轨通信层按约 `360 km` 换算为约 `14.0` 游戏高度，中继层按约 `525 km` 换算为约 `20.4` 游戏高度。场景初始化时直接生成第一批 `10` 颗卫星；之后每 `24` 个火星日直接新增 `10` 颗，第一阶段上限 `120` 颗。每批按两串编队飞过天空，每串 `5` 颗，保持同轨道面和小相位间距，视觉上接近从地表看到的一串星点。卫星模型保持 Starlink V2 Mini 风格的扁平本体、深色相控阵面板和长太阳翼比例；真实比例下本体小于 `0.001` 游戏单位，因此轨道模型采用很小的可见实体代理加圆形微光点表现。当前版本为保证可见性，微光点和实体代理比上一版略增大，但仍避免像近处方形飞行器；雷达/任务 HUD 显示轨道链路状态，不做释放动画。每颗星链卫星都是缩放枪可命中的实体目标，首次命中奖励隐藏发现积分。
- 基地布局：主要建筑和三艘飞船集中在首页亮面，形成“居住舱中心 + 工业/仓储/飞船环绕”的基地组团；三组太阳能电池阵列平行排成一条赤道式电力场，放在首页可见、靠近 03 返回飞船一侧的亮面受光区域，约位于源码坐标 `(-22,52) / (-4,52) / (14,52)`，并远离温室、医疗舱和飞船；坠毁飞船残骸、福福救援点、黑色方碑、足球、NASA Perseverance 火星车和其他小型/神秘物体保持原位置，不纳入主基地搬迁。
- 流星：天空保留三颗流星，主体是不自发光的石质陨石，使用低精度灰黑贴图 `src/assets/meteor-rock-generated.png` 和 `MeshStandardMaterial`，只在受太阳/环境光照时可见；尾迹使用火星车粉尘同源的半透明雾状 sprite 沿运动反方向拖曳，不再使用线段尾巴。两颗远处流星按太阳轨道上的远处天体处理；其中一颗改为绕火星的固定圆形近火轨道，轨道半径恒定为 `PLANET_RADIUS + 230`，从玩家初始基地上空轨道点开始，每现实约 `90` 秒绕行一圈并回到同一上空轨道点。近火流星主体放大到接近飞船量级，不使用头部蓝色气体罩，也不使用明显蓝色自发光，必须直接露出石质陨石本体；仅在本体后方保留更长的蓝色雾状拖尾。近火流星会在雷达上显示为蓝色点。只有这颗近火轨道石质流星本体可被缩放枪锁定和缩放，首次命中奖励隐藏发现积分；另外两颗远处天体不参与缩放枪判断，也不作为雷达重点目标。
- NASA 火星车：地表增加一个静态 `NASA 火星车 Perseverance / Jezero Crater` 纪念点。参考 NASA Perseverance/Jezero 资料，模型表现六轮、摇臂悬挂、桅杆相机、车身仪器箱、RTG 圆筒和机械臂；它是静态真实感地标，不加入基地巡逻车辆系统。
- 语言：首页右上角提供语言切换按钮，默认 `简体中文`。按钮为约 `24px` 的立体小圆球，中心点沿用原右上角位置；球体使用中国/美国国旗 CSS 贴图、明暗高光和缓慢自转表现。中文模式显示美国国旗，点击切换为美式英文界面；英文模式显示中国国旗，可点击切回。进入游戏后隐藏并禁用语言切换，当前局语言沿用首页选择。首页静态文案、HUD、任务、奖励、互动、地图、系统提示和对话舞台通过轻量 i18n 管线刷新；剧情对白和运行时提示优先走 `runtimeEnglishTexts` 精确英文表，避免关键词替换造成半中文半英文。首页“故事概要”跳转到 `/story-overview.html?lang=<当前语言>`，独立故事页会读取 query 或 `localStorage["mars.language"]` 自动切换中英文。
- 首屏防裸 HTML：`index.html` 头部保留极小 critical CSS，提前设置暗色背景、全屏容器和未进入游戏时 HUD 隐藏，避免 Vite dev server 或慢网下 CSS 尚未加载时出现白底黑字闪屏。完整视觉仍由 `src/style.css` 接管。
- 双语检查：新增 UI、任务、对白或标签后运行 `pnpm i18n:audit`。脚本会强检查 `index.html` 的 i18n key 是否中英齐全，并基于 `scripts/i18n-audit-baseline.json` 提醒新增的中文源文本是否还未进入 `i18n`、`exactEnglishTexts`、`runtimeEnglishTexts` 或 `englishPhrasePairs`。当前存量中文已写入 baseline；确认新增中文暂时允许保留为中文权威源时，运行 `pnpm i18n:audit:update` 更新基线。需要让警告阻断时使用 `pnpm i18n:audit:strict`。
- 出室内镜头：从居住舱、温室或飞船内舱出来时，镜头临时切到人物正面，等玩家离开主体建筑遮挡范围后自动恢复跟随视角。
- 飞船升降梯镜头：从飞船内舱出来后，如果玩家启动升降梯下行，出舱正面镜头保持到升降梯到达地面停止，再恢复默认第三人称跟随，避免下行中途突然切镜头。
- 任务提示：桌面和移动端都使用 `V` 右侧的 `R` 任务按钮开关当前任务；首次进入游戏时任务卡短暂展开后自动收起，任务文本更新时 `R` 外框发光，玩家打开后清除提醒。移动端小屏上 `R` 任务面板和 `F` 地图互斥，只允许显示一个；桌面端 `R` 和 `F` 可以同时打开。
- 第三人称防遮挡：室外第三人称下，如果玩家到相机的视线穿过建筑、飞船或其他 `world.colliders` 固体，镜头会沿地表法线平滑抬高，离开遮挡后自动回到原高度，避免角色被居住舱等大型模型完全挡住。
- 建筑碰撞：长条舱体和矩形建筑使用多排圆形足迹 collider 覆盖外壳，而不是单圆或单排圆近似；居住舱、温室、氧气生产站、甲烷燃料厂、车库、科研舱、物资仓和医疗舱都应阻挡 Alex 与足球进入模型内部。进入室内仍通过互动动作完成，不依赖玩家从模型门洞直接走入。
- 飞船升降台：角色站立高度以升降台地板模型的实际顶面为准，并保留少量鞋底余量；不要再只依赖手填 `surfaceY`，否则平台运动和模型缩放会让角色陷入平台。
- 对话框布局：剧情对话舞台使用动态宽度，桌面/宽屏约为屏幕宽度 `70vw`，同时保留左右安全边距；小屏幕切到约 `90vw`，避免文字过窄。小屏幕对话框固定高度并整体上移，不落到底；正文区域最多显示三行，超过三行由运行时自动分页；选项/继续按钮固定在面板底部按钮区，按钮大小和距下边框距离不随正文页内容变化。
- 移动端双选项对话：当小屏对话只有两个选项时，正文自动追加 `A.` / `B.` 选项说明；底部按钮固定为一行两列，只显示短按钮 `A` 和 `B`，避免长选项文字撑高或掉出面板。桌面端继续显示完整选项文案。
- 对话按钮音效：点击或键盘确认对话选项、继续翻页、推进对白或关闭对白时，播放与首页“进入基地”一致的短促 UI beep，增强互动反馈。
- 互动按钮：移动端靠近互动对象时，入口按钮使用居中窄宽度和轻微圆角矩形，避免触碰或遮挡左侧 HUD 文字；桌面端互动选项也统一使用同样的轻圆角按钮语言。
- 顶部圆形按钮：`V`、`R`、`F` 关闭态统一使用半透明深色底；对应信息打开时不再填色、不加粗外圈，而是使用静态红色光晕表示打开。只有 `R` 在有新任务信息时使用动态脉冲发光；玩家打开任务信息后，`R` 停止动态脉冲并保持静态发光，直到关闭任务面板。顶部按钮的焦点态不使用浏览器默认蓝色框，只允许“不发红光 / 静态红光 / 动态红光”三种视觉状态。`V` 只隐藏状态、标签和互动提示，不拦截 `R` 任务面板；即使 HUD 已折叠，玩家点击 `R` 仍应看到当前任务。
- 退出确认：游戏中按 `Esc` 打开确认窗口，默认选中“返回游戏”；切换到“退出游戏”并确认后才返回主页。
- 操作指南：按住 ESC 下方的 `· / ~` 键显示，松开隐藏；玩家使用过一次后，雷达下方的操作指南提示隐藏。
- 神秘代码：输入 `HUYEE` 播放“叮咚”音并切换飞行模式；飞行模式下按住空格上升，按住 `Control/Ctrl` 下降，WASD/方向键继续控制移动。按 `Control/Ctrl` 降到火星地表时自动退出飞行并隐藏喷焰；如果下降到飞船升降台顶部或高处廊道的可行走范围，会落到该平台并切回步行。再次输入 `HUYEE` 也可手动关闭。
- 缩放枪：程序化手持模型默认隐藏；玩家靠近黑色方碑按 `E` 触发对话并点击“获取”后装备到 Alex 右手。拥有后按 `X` 进入/退出瞄准态，中央显示缩放枪准星和锁定目标；瞄准态下 Alex 不能走动，`WASD` 改为全方位控制准星，按 `E` 放大、按 `Q` 缩小，`=` 和 `-` 保留为备用缩放键；效果按原始尺寸限制在约 `0.25x` 到 `4x`，60 秒后自动恢复。太阳也属于可锁定缩放目标，因此缩放枪射程需要覆盖远端太阳实体。
- 相机：首次把足球踢进球门后解锁，不创建手持模型。按 `G` 打开/收起摄像框；取景状态下 Alex 停止移动，`A/D` 转向，`W/S` 调整相机放大倍数，`E/Enter` 把当前 WebGL 画面保存为 PNG 数据并写入居住舱照片墙。renderer 使用 `preserveDrawingBuffer` 保证截图可靠。照片墙位于居住舱内部，一共 `7` 个实体相框；缩略图只做小图展示，靠近照片墙按 `E` 进入大图查看器。大图查看器中 `A/D` 切换照片、`W` 放大、`S` 恢复、`Q` 退出、`E/Enter` 触发当前照片下载；浏览器不静默保存到桌面。
- 奖励 HUD：右上角 `F` 地图按钮左侧显示金币和积分两个圆角矩形按钮，位置不变且不显示中文标签。金币初始显示 `$`，获得后显示三位数字如 `001`，颜色和左侧“基地”标题一致；积分初始显示 `0`，获得后显示三位数字如 `001`，颜色和顶部按钮红色光晕一致。金币/积分按钮的边框、底色、圆角和字体厚度沿用首页语言切换按钮的视觉样式，但保留各自颜色。积分和金币变化时数字弹跳发光，任务积分播放上升调奖励音，金币逐枚拾取时播放“叮”。
- 积分系统：前端运行时维护 `scorePoints`、`awardedEvents` 和建筑/隐藏发现记录。主线任务默认 `+100`，支线任务默认 `+40`，建筑探索默认 `+10`，隐藏问号目标默认 `+30`，搞笑事件默认 `+10`，负面搞笑事件默认 `-10`。建筑探索积分到达范围后自动发放，不显示确认按钮或浮动提示；如果玩家首次撞到建筑、飞船或太阳能阵列等实体碰撞体，也会立即触发对应的一次性探索积分，避免交互点半径和模型阻挡范围不一致导致漏判。神秘物体包括足球、NASA Perseverance 火星车、黑色方碑、未知生命和坠毁/未编号异常物体；未发现前在雷达上用 `?` 标记并只显示“未知物体/未知生命迹象”，首次接近、碰撞或对话发现后奖励隐藏发现积分，随后雷达移除 `?` 并显示真实名称。后续新增任务应优先走统一奖励 API，只有特殊事件才单独覆盖分值。
- 职位系统：人员 HUD 中 Alex 下方显示动态职位，初始为 `实习巡航员`。前端运行时维护 `currentRank`、`completedAnyTask` 和 `rankRules`；完成任意正式任务或积分达到 `100` 后晋级 `初级宇航员`，后续按 `300 / 700 / 1500 / 3000` 积分晋级。职位只升不降，金币不参与晋级；最高称号 `第一位居民` 暂由主线最佳结局条件解锁。
- 随机金币：前端运行时维护单个金币组，每组 10 枚并沿地表切线排成一列，刷新周期为真实游玩时间 4 小时。生成时在星球表面随机抽样，并根据 `world.colliders` 避开建筑、飞船、太阳能阵列、石碑、岩石等固体；长时间无人收取时迁移到新位置，不叠加多组。雷达只显示金币组中心的一个金币标记。
- 火星足球：居住舱外放置一个可碰撞足球。足球使用程序化白色足球贴图、黑色拼块、缝线和 bump 细节；运动采用轻量球面物理，竖直方向使用火星重力 `3.71 m/s²`，地表方向按小星球切线滚动。Alex、机器人、火星车和运输车会按接触速度给足球冲量；足球的踢出冲量和最高速度按偏夸张的游戏手感处理，约为早期版本的 2 倍。建筑、飞船、岩石等 `world.colliders` 固体会让足球反弹并损失部分能量。足球出生点穿过星球直径的另一面放置球门，落点会避开障碍物；足球进入球网后每次奖励 `+100` 积分，并立即回到出生点。首次进球还会一次性解锁相机奖励。

## 部署方案

当前 1.0 的单人体验仍是纯前端静态网页游戏，构建后产物在 `dist/`。多人 v1 额外需要一个常驻 Node WebSocket 服务，用于同步公共大厅中其他玩家的位置和外观状态；该服务不保存任务进度，不需要数据库。
公开故事页图片位于 `public/story-assets/`，线上部署时使用 WebP 版本，避免复制未引用的大 PNG 副本。

推荐部署方式：

- 本地执行 `pnpm build` 或 Codex 内置 pnpm 构建。
- 上传 `dist/` 内容到服务器静态站点目录。
- 使用 Nginx 托管根路径。
- 如启用多人模式，运行 `pnpm dev:server` 或线上 Node 进程启动 `server/multiplayer.ts`，并让 Nginx 将 `/ws` 反向代理到该进程。

详细步骤见：`docs/deployment.md`

## 多人同步方案

多人 v1 采用“客户端权威位置广播 + 服务端房间转发”：

- 公共房间固定为 `public-mars-alpha`。
- 前端身份存储：
  - `localStorage["mars.playerId"]`
  - `localStorage["mars.displayName"]`
- 前端通过 `src/multiplayer.ts` 连接 WebSocket：
  - 开发环境默认：`ws://127.0.0.1:8787/ws`
  - 线上默认：同域名 `/ws`
  - 可用 `VITE_MULTIPLAYER_WS_URL` 覆盖。
- 客户端约每秒 12 次发送本地玩家快照：位置、四元数、速度、飞行状态和室内/室外状态。
- 服务端 `server/multiplayer.ts` 只维护在线玩家列表、心跳超时和状态广播；不判定任务、不写数据库、不保存长期进度。
- 左侧人员 HUD 通过多人客户端的远程玩家列表显示在线人数，最低显示 `1人`。
- 远程玩家复用 `createMarsEngineer()` 生成宇航员模型，并在本地做插值平滑。
- 远程玩家不可碰撞、不可互动，不进入任务系统。
- 当 WebSocket 服务不可用或断开时，前端安静重试，单人游戏继续运行。

## 音频方案

背景音乐使用原 MP3 的压缩版：

- 源文件：`星际低语_no-watermark (1).mp3`，约 10MB，只保留在本地，不提交 GitHub。
- 项目内文件：`src/assets/audio/mars-background-light.mp3`，64kbps，约 2.5MB。

原因：

- 保留原背景音乐的听感。
- 减小部署包体积。
- 保持纯前端静态部署，不增加后端依赖。
